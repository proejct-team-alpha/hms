### 2. ReservationService — cancelReservation 반환 타입 변경

취소 완료 페이지에서 예약 정보를 표시하기 위해 `void` → `ReservationCompleteInfo` 반환으로 변경.
취소 처리 전에 예약 정보를 미리 저장하여 반환한다.

```java
@Transactional
public ReservationCompleteInfo cancelReservation(Long id) {
    Reservation reservation = reservationRepository.findById(id)
            .orElseThrow(() -> new IllegalArgumentException("예약을 찾을 수 없습니다."));

    // 취소 전에 정보 저장
    ReservationCompleteInfo info = new ReservationCompleteInfo(
            reservation.getReservationNumber(),
            reservation.getPatient().getName(),
            reservation.getDepartment().getName(),
            reservation.getDoctor().getStaff().getName(),
            reservation.getReservationDate().toString(),
            reservation.getTimeSlot()
    );

    reservation.cancel();
    return info;
}
```

### 3. ReservationController — 완료 페이지 엔드포인트 추가 및 PathVariable 수정

#### 문제: PathVariable 바인딩 실패

Spring Boot는 컴파일 시 `-parameters` 플래그 없이는 파라미터 이름을 인식하지 못한다.
`@PathVariable Long id`만으로는 `{id}`와 매핑되지 않아 `IllegalArgumentException` 발생.

#### 해결: @PathVariable에 명시적 이름 지정

```java
// 변경 전
@PostMapping("/modify/{id}")
public String modifyReservation(@PathVariable Long id, ...) { }

// 변경 후
@PostMapping("/modify/{id}")
public String modifyReservation(@PathVariable("id") Long id, ...) { }
```

#### 예약 변경 완료 엔드포인트

```java
@PostMapping("/modify/{id}")
public String modifyReservation(@PathVariable("id") Long id,
                                @ModelAttribute ReservationUpdateForm form,
                                RedirectAttributes redirectAttributes) {
    ReservationCompleteInfo info = reservationService.updateReservation(id, form);
    redirectAttributes.addAttribute("reservationNumber", info.getReservationNumber());
    redirectAttributes.addAttribute("name",       info.getPatientName());
    redirectAttributes.addAttribute("department", info.getDepartmentName());
    redirectAttributes.addAttribute("doctor",     info.getDoctorName());
    redirectAttributes.addAttribute("date",       info.getReservationDate());
    redirectAttributes.addAttribute("time",       info.getTimeSlot());
    return "redirect:/reservation/modify-complete";
}

@GetMapping("/modify-complete")
public String modifyComplete(HttpServletRequest request) {
    request.setAttribute("pageTitle", "예약 변경 완료");
    return "reservation/reservation-modify-complete";
}
```

#### 예약 취소 완료 엔드포인트

```java
@PostMapping("/cancel/{id}")
public String cancelReservation(@PathVariable("id") Long id,
                                RedirectAttributes redirectAttributes) {
    ReservationCompleteInfo info = reservationService.cancelReservation(id);
    redirectAttributes.addAttribute("reservationNumber", info.getReservationNumber());
    redirectAttributes.addAttribute("name",       info.getPatientName());
    redirectAttributes.addAttribute("department", info.getDepartmentName());
    redirectAttributes.addAttribute("doctor",     info.getDoctorName());
    redirectAttributes.addAttribute("date",       info.getReservationDate());
    redirectAttributes.addAttribute("time",       info.getTimeSlot());
    return "redirect:/reservation/cancel-complete";
}

@GetMapping("/cancel-complete")
public String cancelComplete(HttpServletRequest request) {
    request.setAttribute("pageTitle", "예약 취소 완료");
    return "reservation/reservation-cancel-complete";
```

### 5. reservation-cancel.mustache — CSRF 토큰 추가

#### 문제: 403 Forbidden 에러

Spring Security의 CSRF 보호 기능으로 인해 POST 요청 시 토큰이 없으면 403 에러 발생.

#### 해결: CSRF 토큰 추가

```html
<form method="post" action="/reservation/cancel/{{id}}" class="flex-1">
  {{! CSRF 토큰 }}
  <input type="hidden" name="{{_csrf.parameterName}}" value="{{_csrf.token}}">
  <button type="submit" class="w-full py-3 bg-red-500 text-white ...">
    예약 취소
  </button>
</form>
```

## 해결한 버그 및 이슈

### 1. PathVariable 바인딩 실패 (IllegalArgumentException)

**증상**:
```
java.lang.IllegalArgumentException: Name for argument of type [java.lang.Long] not specified,
and parameter name information not available via reflection.
Ensure that the compiler uses the '-parameters' flag.
```

**원인**:
Spring Boot는 기본적으로 `-parameters` 플래그 없이 컴파일되므로 파라미터 이름을 리플렉션으로 얻을 수 없다.
`@PathVariable Long id`만으로는 URL의 `{id}`와 매핑되지 않음.

**해결**:
`@PathVariable("id")`로 명시적 이름 지정.

---

### 2. CSRF 토큰 누락 (403 Forbidden)

**증상**:
```json
{
  "success": false,
  "errorCode": "INTERNAL_ERROR",
  "message": "서버 오류가 발생했습니다.",
  "timestamp": "2026-03-11T06:22:55.794546300Z",
  "path": "/error/403"
}
```

**원인**:
Spring Security의 CSRF 보호 기능이 활성화되어 있지만,
`reservation-cancel.mustache`와 `reservation-modify.mustache` 폼에 CSRF 토큰이 없음.

**해결**:
모든 POST 폼에 CSRF 토큰 hidden input 추가.

```html
<input type="hidden" name="{{_csrf.parameterName}}" value="{{_csrf.token}}">
```

---

### 3. Tailwind CSS 클래스 미적용 (투명한 버튼)

**증상**:
예약 조회 페이지의 "조회하기" 버튼이 투명하게 표시됨.

**원인**:
Tailwind CSS가 재빌드되지 않아 새로운 HTML의 클래스가 CSS 파일에 포함되지 않음.

**해결**:
`npm run build:css` 실행하여 Tailwind CSS 재생성.

## 개선 효과

### 1. 일관된 사용자 경험

- 예약 변경 화면이 직접선택예약과 동일한 UI를 제공하여 학습 곡선 감소
- 모든 완료 페이지가 통일된 디자인 패턴을 따름

### 2. 명확한 피드백

- 변경/취소 완료 시 전용 페이지로 이동하여 작업 결과를 명확히 표시
- 색상 코딩 (파란색 = 변경, 빨간색 = 취소)으로 작업 유형 즉시 인식

### 3. 보안 강화

- CSRF 토큰 추가로 Cross-Site Request Forgery 공격 방지
- Spring Security 보안 정책 준수

### 4. 안정성 향상

- PathVariable 바인딩 문제 해결로 500 에러 방지
- 명시적 파라미터 이름 지정으로 코드 가독성 향상

## 참고 사항

### Gradle 데몬 크래시 이슈

개발 중 Gradle 데몬이 예기치 않게 종료되는 현상 발생:

```
FAILURE: Build failed with an exception.
* What went wrong:
Gradle build daemon disappeared unexpectedly (it may have been killed or may have crashed)
```

**해결**: 애플리케이션 재시작 후 정상 동작 확인.

### Tailwind CSS 빌드 시점

새로운 HTML 템플릿이나 클래스를 추가할 때마다 `npm run build:css` 실행 필요.
개발 중에는 `npm run watch:css`로 자동 빌드 설정 가능.
