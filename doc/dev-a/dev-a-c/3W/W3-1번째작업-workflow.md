
## 실행 흐름

```
서버: ItemManagerController → item-list 뷰에 물품 목록 전달
  → 테이블 헤더: '현재 재고' th | '입고' th (신규)
  → 데이터 행:
    - '현재 재고' td: {{quantity}}개 수량만 표시 (text-right)
    - '입고' td: 입고 수량 입력 + 버튼 폼만 표시 (text-center)
  → 빈 행: colspan="7" (기존 6에서 수정)
```

## 실행 흐름에 대한 코드

### 1. 테이블 헤더 — '입고' 컬럼 추가

```html
<!-- '현재 재고' th 다음에 '입고' th 추가 -->
<th class="p-4 font-medium text-slate-700 text-center">현재 재고</th>
<th class="p-4 font-medium text-slate-700 text-center">입고</th>
```

> **💡 입문자 설명**
> - **이 코드가 하는 일**: 테이블의 헤더(제목 행)에 "입고"라는 새 컬럼 제목을 추가합니다. `p-4`는 안쪽 여백, `font-medium`은 글씨 굵기, `text-center`는 가운데 정렬을 의미하는 Tailwind CSS 클래스입니다.
> - **왜 이렇게 썼는지**: 기존 '현재 재고' 컬럼에 수량과 입고 폼이 혼재되어 가독성이 낮았기 때문에, 역할별로 컬럼을 분리해 UI를 명확하게 구분합니다.
> - **쉽게 말하면**: 표의 첫 번째 줄(제목 행)에 "입고"라는 열 제목을 새로 추가하는 코드입니다.

### 2. 테이블 행 — td 분리

```html
<!-- 현재 재고 td — 수량만 표시 -->
<td class="p-4 text-sm text-right">
  <span class="font-bold text-slate-800">{{quantity}}개</span>
</td>

<!-- 입고 td — 폼만 표시 -->
<td class="p-4 text-sm text-center">
  <form method="POST" action="/item-manager/item/restock" class="flex items-center justify-center gap-2">
    <input type="hidden" name="{{_csrf.parameterName}}" value="{{_csrf.token}}">
    <input type="hidden" name="id" value="{{id}}">
    <input type="number" name="amount" min="1" class="w-16 px-2 py-1 border rounded text-center">
    <button type="submit" class="px-3 py-1 bg-blue-600 text-white rounded text-xs">입고</button>
  </form>
</td>
```

> **💡 입문자 설명**
> - **이 코드가 하는 일**: 테이블의 각 데이터 행(`<td>`)을 두 개로 분리합니다. 첫 번째 `<td>`는 재고 수량 숫자만 보여주고, 두 번째 `<td>`는 입고 수량 입력 폼만 담습니다. `{{quantity}}`는 서버에서 전달한 재고 수량 값입니다.
> - **왜 이렇게 썼는지**: 수량 표시와 입고 폼을 같은 셀에 넣으면 레이아웃이 복잡해집니다. 역할에 따라 셀을 분리하면 각각 독립적으로 스타일을 적용할 수 있습니다.
> - **쉽게 말하면**: 하나의 표 칸에 숫자와 버튼이 함께 있던 것을 "숫자 칸"과 "버튼 칸"으로 나눈 것입니다.

### 3. 빈 행 colspan 수정

```html
<!-- colspan 6 → 7 -->
<td colspan="7" class="p-8 text-center text-slate-400 text-sm">
  등록된 물품이 없습니다.
</td>
```

> **💡 입문자 설명**
> - **이 코드가 하는 일**: 물품이 없을 때 표시하는 메시지가 들어있는 셀이 테이블 전체 너비를 차지하도록 `colspan` 값을 7로 수정합니다.
> - **왜 이렇게 썼는지**: 컬럼이 6개에서 7개로 늘었기 때문에 빈 행의 `colspan`도 7로 맞춰야 메시지가 전체 너비에 깔끔하게 표시됩니다. 숫자가 맞지 않으면 레이아웃이 깨집니다.
> - **쉽게 말하면**: 표의 열 수가 6개에서 7개로 늘었으니, "데이터 없음" 안내 메시지도 7칸을 합친 너비로 표시하도록 수정한 것입니다.
