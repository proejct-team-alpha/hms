### 3. 빈 행 colspan 수정

- `colspan="6"` → `colspan="7"` (컬럼 수 증가 반영)

> **💡 입문자 설명**
>
> **왜 '현재 재고'와 '입고'를 한 컬럼에서 두 컬럼으로 분리했는지**
> - 기존에는 하나의 `<td>` 안에 재고 수량 표시와 입고 폼이 함께 있어 레이아웃이 복잡했습니다. 역할이 다른 두 요소를 분리하면 각각 정렬을 독립적으로 제어할 수 있고, 읽기도 더 쉬워집니다.
> - **컨트롤러/서비스 변경이 없는 이유**: 이 작업은 순수하게 HTML 구조(View) 변경입니다. 서버에서 보내는 데이터는 그대로이고, 화면에 표시하는 방식만 바꿨습니다. MVC 패턴에서 View만 수정한 경우입니다.
>
> **`colspan` 수정이 필요한 이유**
> - 테이블 하단에 "데이터가 없습니다" 같은 메시지를 표시할 때 `colspan`으로 전체 컬럼을 가로질러 표시합니다. 컬럼 수가 6→7로 늘었으므로 `colspan="6"` → `colspan="7"`로 맞춰야 레이아웃이 깨지지 않습니다.
>
> **쉽게 말하면**: 표의 "재고 현황" 열을 "현재 보유량"과 "입고 처리" 두 칸으로 나눈 것입니다. 서버 로직은 그대로고 화면 표시 방식만 바꿨습니다.
