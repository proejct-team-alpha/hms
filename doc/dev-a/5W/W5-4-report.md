## 변경 내용 상세

### reservation-modify.mustache

#### 1. 현재 예약 날짜·시간 변수 주입

```javascript
// 현재 예약의 날짜·시간 (서버에서 주입) — 동일 날짜 선택 시 해당 슬롯 선택 불가 처리
const currentReservationDate = '{{reservationDate}}';
const currentTimeSlot = '{{timeSlot}}';
```

> **💡 입문자 설명**
> - **이 코드가 하는 일**: 서버가 Mustache 템플릿을 렌더링할 때 현재 예약의 날짜(`reservationDate`)와 시간(`timeSlot`) 값을 JavaScript 변수로 주입합니다. 이 값은 이후 슬롯 비활성화 로직에서 "현재 예약과 동일한 슬롯인지" 비교하는 데 사용됩니다.
> - **왜 이렇게 썼는지**: JavaScript는 서버의 Java 객체에 직접 접근할 수 없으므로, 서버 렌더링 단계에서 Mustache `{{변수명}}`을 통해 값을 HTML 안에 심어두는 방법을 사용합니다. 작은따옴표(`' '`)로 감싸 문자열 변수로 선언합니다.
> - **쉽게 말하면**: 서버가 페이지를 만들 때 "이 사람의 현재 예약은 3월 20일 10시야"라고 페이지 안에 적어두면, 브라우저 JS가 그 값을 읽어 해당 슬롯을 회색으로 표시합니다.

#### 2. datePicker.onChange 슬롯 렌더링 로직 변경

| 조건 | 변경 전 | 변경 후 |
|------|---------|---------|
| 오늘 날짜 + 지난 시간 | 선택 가능 | 회색 글자 + 선택 불가 |
| 현재 예약 날짜 + 현재 예약 시간 | 선택 가능 | 회색 글자 + 선택 불가 |
| 내일 이후 날짜 | 선택 가능 | 정상 유지 |
| booked 슬롯 | "(예약불가)" | 유지 |

#### 슬롯 판단 우선순위

```
1. booked 슬롯          → 회색 + "(예약불가)" + disabled
2. 현재 예약 날짜+시간  → 회색 + disabled (텍스트 변경 없음)
3. 오늘 날짜 + 지난시간 → 회색 + disabled (텍스트 변경 없음)
4. 선택 가능 슬롯       → 정상 표시
```

> **💡 입문자 설명**
> - **이 코드가 하는 일**: 시간 슬롯 드롭다운의 각 옵션을 어떻게 표시할지 결정하는 4가지 우선순위 규칙을 정의합니다. 위 순서대로 검사해 처음 해당하는 규칙을 적용합니다.
> - **왜 이렇게 썼는지**: 우선순위가 필요한 이유는 한 슬롯이 여러 조건에 동시에 해당될 수 있기 때문입니다. 예를 들어 "현재 예약 시간인 동시에 지난 시간"인 슬롯은 2순위(현재 예약) 규칙이 먼저 적용됩니다. `disabled`는 HTML 속성으로 드롭다운 옵션을 선택 불가 상태로 만들고, `text-slate-400`은 TailwindCSS의 회색 글자 클래스입니다.
> - **쉽게 말하면**: 시간표의 각 칸을 "다른 사람 예약", "내 현재 예약", "지난 시간", "예약 가능" 네 가지 색으로 구분하는 규칙입니다.
