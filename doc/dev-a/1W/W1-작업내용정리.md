# [입문자용 가이드] 1주차: 예약 화면의 첫 걸음

이 문서는 처음 코드를 만져보는 분들을 위해 1주차 작업 내용(예약방법 선택 화면 URL 연결)을 아주 쉽고 자세하게 설명합니다.

---

## 💡 1. 무엇을 만드는 건가요? (작업 목표)

우리는 지금 병원의 예약 시스템을 만들고 있습니다. 
사용자가 "예약하기" 버튼을 눌렀을 때, 가장 먼저 **어떤 방식으로 예약할지(AI 추천 예약 vs 직접 선택 예약) 묻는 화면**이 나와야 합니다.

이번 작업의 목표는 사용자가 주소창에 `/reservation` 이라고 입력했을 때, 이 선택 화면이 예쁘게 뜨도록 서버(Controller)와 화면(Mustache)을 연결해 주는 것입니다.

---

## 🔑 2. 알아두면 좋은 필수 용어

이 코드를 이해하기 위해 딱 3가지만 기억해 주세요!

1.  **Controller (컨트롤러)**: 병원의 "안내데스크" 역할을 합니다. 사용자가 브라우저(크롬 등)에서 어떤 주소를 입력하면, 이 컨트롤러가 요청을 가장 먼저 받아서 "아, 예약 화면을 보여달라고? 알겠어!" 하고 화면을 찾아줍니다.
2.  **Mustache (머스타치)**: 서버에서 만든 데이터를 HTML 화면에 예쁘게 끼워 넣어주는 템플릿(틀) 엔진입니다. `.mustache` 확장자를 가집니다.
3.  **@GetMapping ("주소")**: 컨트롤러에게 "만약 사용자가 이 주소로 들어오면, 이 아래에 있는 코드를 실행해!"라고 알려주는 이름표(어노테이션)입니다.

---

## 🌊 3. 전체 동작 흐름도

사용자가 예약을 하려고 할 때 우리 프로그램은 이렇게 움직입니다.

```text
[사용자] 브라우저에 접속 (GET /reservation)
   ↓
[서버] ReservationController 대기 중...
   ↓ "@GetMapping이 붙은 곳으로 왔네? 내가 처리할게!"
[서버] 화면에 넘겨줄 데이터를 준비 (예: 페이지 제목 = "진료 예약")
   ↓
[서버] "reservation/patient-choice" 라는 이름의 Mustache 화면 파일 선택!
   ↓
[화면] HTML로 예쁘게 그려진 후 사용자 브라우저에 표시됨
```

## 💻 4. 핵심 코드 아주 자세히 살펴보기

자, 이제 이 흐름을 코드로 어떻게 적었는지 한 줄 한 줄 뜯어보겠습니다.

### 4.1. 서버의 안내데스크: ReservationController.java

이 코드는 사용자가 접속했을 때 화면을 찾아주는 역할을 합니다.

```java
// Spring 설정: "이 클래스는 사용자 요청을 받는 안내데스크(Controller)야!" 라고 알려줍니다.
@Controller 
// 기본 주소 설정: "이 아래에 있는 모든 코드는 /reservation 으로 시작하는 주소들만 처리해!"
@RequestMapping("/reservation") 
public class ReservationController {

    // 1. @GetMapping(""): 사용자가 덧붙이는 주소 없이 딱 '/reservation'까지만 입력하고 들어올 때 이 메서드가 실행됩니다.
    // 만약 @GetMapping("/detail") 이었다면 '/reservation/detail'로 들어와야 실행됩니다.
    @GetMapping("")
    
    // 2. HttpServletRequest request: 사용자의 요청 정보를 담고 있는 '장바구니' 같은 객체입니다. 
    // 우리가 화면(Mustache)에 전달하고 싶은 데이터가 있으면 이 장바구니에 담아주면 됩니다.
    public String patientChoice(HttpServletRequest request) {
        
        // 3. 장바구니에 데이터 담기
        // 화면 쪽에 "pageTitle"이라는 이름표를 붙여서 "진료 예약"이라는 글자를 담아 보냅니다.
        // 나중에 화면을 그릴 때 이 글자를 꺼내서 웹 브라우저 탭의 제목으로 씁니다.
        request.setAttribute("pageTitle", "진료 예약");
        
        // 4. 어떤 화면 파일을 보여줄지 결정하기
        // 여기서 "reservation/patient-choice" 라고 이름을 적어서 반환(return)하면,
        // Spring Boot가 알아서 'src/main/resources/templates/reservation/patient-choice.mustache' 파일을 찾아 사용자에게 보여줍니다.
        // 확장자(.mustache)는 생략해도 알아서 찾아줍니다!
        return "reservation/patient-choice";
    }
}
```

### 4.2. 사용자가 보는 화면: patient-choice.mustache

컨트롤러가 찾아준 이 화면 파일에는 예약 방식을 선택하는 버튼들이 있습니다. 버튼을 누르면 다른 주소로 이동하도록 만들었습니다.

```html
<!-- ... (위쪽 HTML 코드 생략) ... -->

<div class="button-group">
    <!-- 1. 처음으로 버튼 -->
    <!-- onclick="location.href='주소'" 는 버튼을 클릭했을 때 브라우저를 해당 주소로 이동시켜 달라는 자바스크립트 명령어입니다. -->
    <!-- '/' 는 우리 홈페이지의 가장 첫 화면(메인 화면)을 의미합니다. -->
    <button onclick="location.href='/'">처음으로 돌아가기</button>

    <!-- 2. AI 증상 추천 예약 버튼 -->
    <!-- 이 버튼을 누르면 서버의 '/reservation/symptom-reservation' 주소로 이동하여, AI 추천을 담당하는 다른 컨트롤러 코드가 실행됩니다. -->
    <button onclick="location.href='/reservation/symptom-reservation'">AI 증상 추천 예약</button>

    <!-- 3. 직접 선택 예약 버튼 -->
    <!-- 이 버튼을 누르면 환자가 직접 진료과와 의사를 고르는 '/reservation/direct-reservation' 주소로 이동합니다. -->
    <button onclick="location.href='/reservation/direct-reservation'">직접 선택 예약</button>
</div>

<!-- ... (아래쪽 HTML 코드 생략) ... -->
```

## ✅ 5. 무엇이 달라졌나요? (테스트해보기)

이 코드를 작성하고 서버를 켜면 어떻게 변하는지 확인해보세요.

1.  인터넷 브라우저를 열고 `http://localhost:8080/reservation` 주소로 접속해 봅니다.
2.  에러 화면(White Label Error Page) 대신, 우리가 만든 **예약 방법 선택 화면**이 예쁘게 나타납니다!
3.  "AI 증상 추천 예약" 버튼이나 "직접 선택 예약" 버튼을 누르면 주소창의 주소가 해당 경로로 정상적으로 바뀌는 것을 볼 수 있습니다.
