# 1주차 전체 정리 — Java 입문자를 위한 설명

> 이 파일은 Spring Boot를 처음 접하는 입문자가 1주차 작업 내용을 이해할 수 있도록 작성되었습니다.

---

## 1주차 핵심 목표

> **"URL을 입력하면 화면이 뜨게 만들기"**

사용자가 브라우저에서 `/reservation` 주소를 입력했을 때 예약 방법 선택 화면이 보여야 한다.
이 단계에서 배우는 것: **URL → Controller → 화면(Mustache)** 로 이어지는 기본 흐름.

---

## 핵심 개념 설명

### 1. Controller (컨트롤러)

Spring Boot에서 **URL 요청을 받아서 처리하는 역할**을 하는 클래스입니다.

```java
@Controller
public class ReservationController {

    @GetMapping("/reservation")
    public String reservationHome() {
        return "reservation/index";  // templates/reservation/index.mustache 를 보여줘라
    }
}
```

- `@Controller`: "이 클래스는 웹 요청을 처리한다"고 Spring에게 알리는 표시
- `@GetMapping("/reservation")`: 브라우저에서 `/reservation` 주소로 GET 요청이 오면 이 메서드를 실행
- `return "reservation/index"`: `src/main/resources/templates/reservation/index.mustache` 파일을 화면에 표시

### 2. Mustache (머스태치)

HTML 파일에 **서버에서 보내준 데이터를 끼워 넣는** 템플릿 엔진입니다.

```html
<!-- templates/reservation/index.mustache -->
<h1>안녕하세요, {{name}}님!</h1>
```

- `{{name}}`: 컨트롤러에서 `model.addAttribute("name", "홍길동")` 으로 전달한 값이 들어감
- 화면에는 `안녕하세요, 홍길동님!` 이 표시됨

### 3. Spring MVC 기본 흐름

```
브라우저 → URL 요청
    ↓
Controller (요청 받음)
    ↓
Model (데이터 준비)
    ↓
Mustache 템플릿 (화면 그리기)
    ↓
브라우저 (완성된 HTML 표시)
```

### 4. @GetMapping vs @PostMapping

| 어노테이션 | 언제 사용 | 예시 |
|-----------|----------|------|
| `@GetMapping` | 페이지를 **조회**할 때 (주소창에서 접근) | 예약 화면 보여주기 |
| `@PostMapping` | 데이터를 **전송**할 때 (form 제출) | 예약 정보 저장하기 |

## 1주차 작업 내용

### 작업: 예약 화면의 첫 걸음

**무엇을 만들었나:**
`/reservation` URL에 접근하면 예약 방법을 선택하는 화면을 보여줌
- "AI 증상 기반 예약" 카드
- "직접 선택 예약" 카드

**핵심 파일:**
- `ReservationController.java` — URL과 화면을 연결하는 컨트롤러
- `templates/reservation/index.mustache` — 화면 HTML

**왜 이게 중요한가:**
모든 웹 기능의 시작점. URL → Controller → View(화면) 흐름을 이해하면 나머지 모든 기능을 이해할 수 있다.

## 입문자가 꼭 기억할 것

| 개념 | 한 줄 설명 |
|------|-----------|
| `@Controller` | URL 요청을 처리하는 클래스 선언 |
| `@GetMapping` | 특정 URL의 GET 요청을 이 메서드가 처리 |
| `return "뷰이름"` | templates/ 폴더의 해당 mustache 파일을 화면에 표시 |
| `model.addAttribute` | 화면에 전달할 데이터를 담는 방법 |
| `{{변수명}}` | Mustache에서 데이터를 출력하는 방법 |
