# [2주차-1] 예약 정보 입력 화면(Form) 완성하기

이 문서는 환자가 이름, 전화번호 등을 입력하는 화면을 서버와 연결하는 과정을 담고 있습니다.

---

## 🧭 1. 워크플로우(Workflow): 어떤 계획을 세웠나요?
*   **목적**: 화면에 있는 '예약하기' 버튼을 눌렀을 때, 입력한 정보들이 서버로 날아가게 하자!
*   **설계**: 
    1. HTML의 `<form>` 태그에 `action="/reservation/create"`와 `method="POST"`를 적어주자.
    2. 각 입력칸(`input`)에 서버가 알아볼 수 있는 이름표(`name`)를 붙여주자.
    3. 보안을 위해 '일회용 비밀번호(CSRF 토큰)'를 숨겨서 같이 보내자.

---

## 🏗️ 2. 리포트(Report): 실제로 어떻게 구현했나요?
계획대로 코드를 수정하여 서버가 데이터를 받을 준비를 마쳤습니다.

### 2.1. 서버의 안내데스크 (Controller)
사용자가 주소를 치고 들어왔을 때 화면을 보여주는 코드를 아주 자세히 설명합니다.

```java
// 1. @GetMapping: 사용자가 "직접 예약 화면 좀 보여줘!"라고 요청할 때 실행됩니다.
@GetMapping("/direct-reservation")
public String directReservation(HttpServletRequest request) {
    
    // 2. 화면 상단에 표시될 제목을 "pageTitle"이라는 이름으로 바구니에 담습니다.
    request.setAttribute("pageTitle", "직접 선택 예약");
    
    // 3. 'reservation/direct-reservation.mustache' 파일을 찾아 사용자에게 보여줍니다.
    return "reservation/direct-reservation";
}
```

### 2.2. 사용자가 보는 화면 (Mustache)
데이터를 묶어서 보내는 '봉투'인 Form 태그를 아주 꼼꼼하게 주석과 함께 살펴봅시다.

```html
<!-- 1. action: 데이터가 도착할 서버 주소 / method: 데이터를 숨겨서 보내는 방식(POST) -->
<form action="/reservation/create" method="POST">

    <!-- 2. 보안 토큰: "나는 해커가 아니야!"라는 증명서입니다. (CSRF) -->
    <!-- 화면에는 안 보이지만(hidden), 서버는 이 값이 있어야만 데이터를 받아줍니다. -->
    <input type="hidden" name="{{_csrf.parameterName}}" value="{{_csrf.token}}">

    <!-- 3. 이름표(name): 서버가 이 데이터를 식별하는 기준입니다. -->
    <label>성함</label>
    <!-- name="name" 이라고 적어야 서버가 "아, 이게 이름이구나!" 하고 인식합니다. -->
    <input type="text" name="name" placeholder="성함을 입력하세요">

    <label>연락처</label>
    <!-- name="phone" 이라고 이름표를 붙였습니다. -->
    <input type="text" name="phone" placeholder="연락처를 입력하세요">

    <button type="submit">예약 신청하기</button>
</form>
```

---

## 💡 3. 왜 이 코드를 사용했고, 어떤 기능을 하나요?
*   **`<form>` 태그**: 여러 개의 입력값(이름, 번호 등)을 **하나의 덩어리**로 묶어서 서버로 전송하는 기능을 합니다.
*   **`name` 속성**: 서버의 자바(Java) 코드에서 데이터를 꺼낼 때 사용하는 **'열쇠'** 역할을 합니다. 이게 없으면 서버는 빈 상자를 받는 것과 같습니다.
*   **CSRF 토큰**: 웹사이트 간 위조 요청 공격을 막기 위한 **보안 장치**입니다. 스프링 시큐리티를 사용하는 환경에서는 선택이 아닌 필수입니다!
