# [2주차-3] 저장된 예약 정보 보여주기 (조회 화면)

이 문서는 DB에 저장된 데이터를 다시 꺼내서 사용자 화면에 예쁘게 보여주는 과정을 설명합니다.

---

## 🧭 1. 워크플로우(Workflow): 어떤 계획을 세웠나요?
*   **목적**: "예약이 잘 되었어요!"라는 안내와 함께 예약 번호, 일시 등을 보여주자!
*   **설계**: 
    1. 저장 직후 `id` 값을 쿼리 파라미터(`?id=123`)로 전달하자.
    2. 컨트롤러에서 이 `id`를 받아 DB에서 정보를 찾아오자.
    3. 찾아온 정보를 Mustache 화면에 끼워 넣어 렌더링하자.

---

## 🏗️ 2. 리포트(Report): 실제로 어떻게 구현했나요?
코드로 데이터를 화면에 뿌려주는 과정을 주석과 함께 봅시다.

### 2.1. 안내데스크의 조회 업무: ReservationController.java
저장된 ID를 가지고 DB 창고에서 정보를 가져옵니다.

```java
// 1. @GetMapping: "예약 완료 화면 보여줘!"라고 요청할 때 실행됩니다.
@GetMapping("/complete")
public String reservationComplete(@RequestParam Long id, HttpServletRequest request) {
    
    // 2. 서비스 호출: "주방장님, 방금 저장된 id가 123번인 예약 좀 찾아봐줘!"
    ReservationInfoDto info = reservationService.getReservationInfo(id);

    // 3. 장바구니 담기: 찾아온 정보를 "model"이라는 이름으로 화면에 넘겨줍니다.
    // Mustache는 이제 이 "model" 안에 있는 정보를 꺼내서 쓸 수 있습니다.
    request.setAttribute("model", info);
    
    // 4. 'reservation/reservation-complete.mustache' 파일을 보여줍니다.
    return "reservation/reservation-complete";
}
```

### 2.2. 사용자 화면: reservation-complete.mustache
서버가 넘겨준 데이터를 중괄호 2개(`{{ }}`)를 사용해 화면에 표시합니다.

```html
<div class="result-box">
    <h2>예약이 성공적으로 접수되었습니다!</h2>
    
    <!-- 1. 서버에서 보낸 "model" 객체 안에 있는 예약 번호를 출력합니다. -->
    <p>예약 번호: <strong>{{model.reservationNumber}}</strong></p>
    
    <!-- 2. 예약자 성함을 출력합니다. -->
    <p>성함: {{model.patientName}} 님</p>
    
    <!-- 3. 예약 날짜를 예쁘게 보여줍니다. -->
    <p>날짜: {{model.reservationDate}}</p>
    
    <button onclick="location.href='/'">홈으로 돌아가기</button>
</div>
```

---

## 💡 3. 왜 이 코드를 사용했고, 어떤 기능을 하나요?
*   **`@RequestParam`**: 주소창의 `?id=123` 부분에서 `123`이라는 **데이터만 쏙 뽑아낼 때** 사용합니다.
*   **DTO (ReservationInfoDto)**: DB 엔티티(실제 DB 테이블과 1:1인 객체)를 직접 화면에 내보내는 것은 보안상 위험할 수 있습니다. 그래서 **화면에 보여줄 정보만 따로 담은** DTO를 사용합니다.
*   **`{{ }}` (Mustache 문법)**: 자바 서버에서 전달한 데이터를 HTML 사이사이에 **동적으로 끼워 넣어주는** 문법입니다.
