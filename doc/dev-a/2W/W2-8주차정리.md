# [2주차-8] 내 예약 취소하기 (상태 변경 기능 구현)

이 문서는 예약된 정보를 삭제하지 않고, "취소됨" 상태로 바꾸어 안전하게 관리하는 과정을 설명합니다.

---

## 🧭 1. 워크플로우(Workflow): 어떤 계획을 세웠나요?
*   **목적**: 환자가 예약을 취소하고 싶을 때, DB에서 해당 예약의 상태를 '취소'로 바꾸자!
*   **설계**: 
    1. 취소 버튼을 누르면 서버에 "이 번호의 예약을 취소해줘!"라고 요청하자.
    2. 서비스(Service)가 DB에서 해당 예약 정보를 찾아 상태를 `CANCELLED`로 변경하자.
    3. 작업이 끝나면 "취소 완료" 화면을 보여주자.

---

## 🏗️ 2. 리포트(Report): 실제로 어떻게 구현했나요?
자바 코드로 예약 상태를 안전하게 취소(Delete 대신 Update)하는 과정을 봅시다.

### 2.1. 안내데스크의 취소 접수: ReservationController.java
취소할 예약 번호를 받아 서비스에 취소 처리를 맡깁니다.

```java
@PostMapping("/cancel/{id}")
public String cancelReservation(@PathVariable Long id, RedirectAttributes rttr) {
    
    // 1. 서비스 호출: "주방장님, 이 번호(id)의 예약 좀 취소 처리해주세요!"
    reservationService.cancelReservation(id);

    // 2. 일시적인 메시지 전달: 다음 화면에 "취소되었습니다"라고 한 번만 보여줄 메시지를 담습니다.
    rttr.addFlashAttribute("message", "예약이 성공적으로 취소되었습니다.");
    
    // 3. 다시 메인 화면으로 돌려보냅니다.
    return "redirect:/";
}
```

### 2.2. 주방장의 상태 변경: ReservationService.java
DB에서 정보를 찾아 '상태'만 쏙 바꾸는 핵심 로직입니다.

```java
@Transactional // 수정을 할 때는 이 안전벨트가 반드시 필요합니다!
public void cancelReservation(Long id) {

    // 1. 취소할 예약 정보를 DB에서 찾아옵니다.
    Reservation reservation = reservationRepository.findById(id)
            .orElseThrow(() -> new CustomException(ErrorType.RESERVATION_NOT_FOUND));

    // 2. 예약 상태를 CANCELLED(취소됨)로 바꿉니다. 
    // 실제 데이터를 삭제(DELETE)하지 않는 이유는 "누가 예약을 취소했는지" 기록을 남기기 위해서입니다.
    reservation.cancel(); 
    
    // 3. 더티 체킹에 의해 메서드가 끝날 때 DB의 상태값이 "취소"로 업데이트됩니다.
}
```

---

## 💡 3. 왜 이 코드를 사용했고, 어떤 기능을 하나요?
*   **Soft Delete (소프트 삭제)**: 데이터를 아예 지우는 것이 아니라, **상태만 바꿔서 지워진 것처럼 처리**하는 방식입니다. 로그 기록이나 나중에 환자가 "내가 취소했었나?" 하고 물어볼 때 확인하기 좋습니다.
*   **`RedirectAttributes`**: 페이지가 바뀐 뒤에 사용자에게 **"취소되었습니다!"**라고 짧게 알림을 띄우고 싶을 때 사용하는 임시 데이터 전송 상자입니다.
*   **보안 필수**: 취소 기능을 만들 때는 **"정말 본인이 취소하는 게 맞는지"** 확인하는 과정(예: 예약 번호 대조)이 서비스 계층에 반드시 포함되어야 합니다.
