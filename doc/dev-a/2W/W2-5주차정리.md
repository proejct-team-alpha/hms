# [2주차-5] 내 예약 정보 바꾸기 (수정 기능 구현)

이 문서는 이미 예약된 정보를 새로운 날짜나 정보로 안전하게 바꾸는 과정을 설명합니다.

---

## 🧭 1. 워크플로우(Workflow): 어떤 계획을 세웠나요?
*   **목적**: 예약한 날짜나 정보를 바꾸고 싶을 때, 새 정보를 입력받아 수정하자!
*   **설계**: 
    1. 수정 버튼을 누르면 기존 정보가 미리 채워진 수정 화면(`reservation-modify.mustache`)을 보여주자.
    2. 수정된 데이터를 서버로 보내면, 서비스(Service)가 DB에 있는 기존 정보를 업데이트하게 하자.
    3. 수정이 끝나면 다시 예약 상세 화면으로 돌아가게 하자.

---

## 🏗️ 2. 리포트(Report): 실제로 어떻게 구현했나요?
자바 코드로 데이터를 수정(Update)하는 과정을 자세히 봅시다.

### 2.1. 기존 정보를 수정 폼에 채우기: ReservationController.java
수정 화면에 들어갈 때, 아무것도 없는 빈칸이 아니라 "내가 예약했던 이전 정보"를 먼저 보여줍니다.

```java
@GetMapping("/modify/{id}")
public String modifyForm(@PathVariable Long id, HttpServletRequest request) {
    
    // 1. 수정할 예약 번호(id)로 기존 정보를 DB에서 가져옵니다.
    ReservationInfoDto info = reservationService.getReservationInfo(id);

    // 2. 화면에 "model"이라는 이름으로 넘겨줍니다. 
    // 그러면 화면의 입력칸들이 이 정보들로 미리 채워집니다.
    request.setAttribute("model", info);
    
    return "reservation/reservation-modify";
}
```

### 2.2. 진짜로 데이터 고치기: ReservationService.java
DB에 있는 정보를 찾아 내용을 업데이트하는 주방장의 요리입니다.

```java
@Transactional // 수정을 할 때는 이 안전벨트가 반드시 필요합니다!
public void updateReservation(Long id, ReservationUpdateForm form) {

    // 1. 수정할 예약 데이터를 DB에서 찾아옵니다. (기존 데이터)
    Reservation reservation = reservationRepository.findById(id)
            .orElseThrow(() -> new CustomException(ErrorType.RESERVATION_NOT_FOUND));

    // 2. 새로운 데이터(form)로 기존 예약의 내용을 덮어씁니다.
    // 여기서는 진료과(Department)를 새로 찾아서 넣어주는 등의 작업을 합니다.
    reservation.changeInfo(form.getNewDate(), form.getNewDept());
    
    // 3. 별도의 save()를 안 해도 메서드가 끝날 때 DB에 짠! 하고 반영됩니다. (더티 체킹)
}
```

---

## 💡 3. 왜 이 코드를 사용했고, 어떤 기능을 하나요?
*   **`@PathVariable`**: 주소창의 `/modify/123` 부분에서 `123`이라는 **예약 고유 번호를 경로에서 바로 추출**할 때 사용합니다.
*   **Dirty Checking (더티 체킹)**: "더러워진(변경된) 것을 체크한다"라는 뜻입니다. 객체의 값이 바뀌었는지만 보고도 스프링이 **자동으로 DB에 업데이트 명령**을 내려주는 아주 똑똑한 기능입니다.
*   **사용자 편의성**: 수정 폼에 기존 데이터를 미리 채워주는 것은 **사용자가 이전 정보를 보면서 고칠 수 있게 배려**하는 중요한 UI/UX 기능입니다.
