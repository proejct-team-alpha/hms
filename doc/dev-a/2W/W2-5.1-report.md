# W2-#5.1 Report: 예약번호 발급, 예약 조회·변경 화면

## 작업 개요

예약 생성 시 `RES-YYYYMMDD-XXX`(당월 누적) 형식의 예약번호를 발급하고,
예약번호 또는 이름+전화번호로 예약을 조회할 수 있는 화면을 제공한다.

> **💡 입문자 설명**

> - **쉽게 말하면**: "이번 달 1일부터 말일까지 예약이 몇 건 있나요?"라고 DB에 묻는 카운터입니다.

### 2. ReservationInfoDto (신규 생성)

조회 결과를 뷰로 전달하기 위한 DTO. `LazyInitializationException` 방지용.
상태를 한글로 변환하고, `RESERVED` 상태만 변경 가능한 `canModify` 플래그 제공.

```java
@Getter
public class ReservationInfoDto {
    private final Long id;
    private final String reservationNumber;
    private final String patientName;
    private final String departmentName;
    private final String doctorName;
    private final String reservationDate;
    private final String timeSlot;
    private final String status;      // 한글 변환 (예약완료 / 접수완료 / 진료완료 / 취소됨)
    private final boolean canModify;  // RESERVED 상태만 true

    public ReservationInfoDto(Reservation r) {
        this.id = r.getId();
        this.reservationNumber = r.getReservationNumber();
        this.patientName = r.getPatient().getName();
        this.departmentName = r.getDepartment().getName();
        this.doctorName = r.getDoctor().getStaff().getName();
        this.reservationDate = r.getReservationDate().toString();
        this.timeSlot = r.getTimeSlot();
        this.status = toKorean(r.getStatus());
        this.canModify = r.getStatus() == ReservationStatus.RESERVED;
    }

    private static String toKorean(ReservationStatus s) {
        return switch (s) {
            case RESERVED  -> "예약완료";
            case RECEIVED  -> "접수완료";
            case COMPLETED -> "진료완료";
            case CANCELLED -> "취소됨";
        };
    }
}
```

> **💡 입문자 설명**
> - **이 코드가 하는 일**: 예약 조회 결과를 화면에 전달하기 위한 DTO입니다. 상태를 한글로 변환하고, 변경 가능한 상태(`RESERVED`)인 경우만 `canModify = true`를 설정합니다.
> - **왜 이렇게 썼는지**: `LazyInitializationException`을 방지하기 위해 엔티티 대신 DTO로 변환하여 뷰에 전달합니다. `switch` 표현식으로 각 상태 코드를 한글로 변환합니다. `canModify`는 Mustache 템플릿에서 `{{#canModify}}` 블록으로 버튼 표시 여부를 제어하는 데 사용됩니다.
> - **쉽게 말하면**: 예약 내역을 보기 좋게 정리한 요약서입니다. 상태를 영어 코드 대신 한글로 표시하고, 변경 가능한 예약에만 "변경/취소" 버튼을 보여주는 플래그를 포함합니다.

### 4. ReservationService — 예약번호 생성 + 조회 메서드 DTO 반환으로 수정

#### generateReservationNumber() — 당월 누적 카운트 기반 예약번호 생성

```java
private String generateReservationNumber() {
    LocalDateTime now = LocalDateTime.now();
    LocalDateTime startOfMonth = now.withDayOfMonth(1).withHour(0).withMinute(0).withSecond(0).withNano(0);
    LocalDateTime endOfMonth = startOfMonth.plusMonths(1).minusNanos(1);
    long count = reservationRepository.countByCreatedAtBetween(startOfMonth, endOfMonth) + 1;
    String dateStr = now.format(DateTimeFormatter.ofPattern("yyyyMMdd"));
    return String.format("RES-%s-%03d", dateStr, count);
}
```

> **💡 입문자 설명**
> - **이 코드가 하는 일**: 현재 날짜와 이번 달 누적 예약 수를 기반으로 `RES-20260320-001` 형식의 예약번호를 생성합니다.
> - **왜 이렇게 썼는지**: `withDayOfMonth(1).withHour(0)...`으로 이번 달 첫 날 자정을 계산하고, `plusMonths(1).minusNanos(1)`로 이번 달 마지막 순간을 계산합니다. `String.format("RES-%s-%03d", ...)`에서 `%03d`는 숫자를 3자리로 맞추고 앞을 0으로 채웁니다(예: 1→001).
> - **쉽게 말하면**: "오늘은 2026년 3월 20일이고, 이번 달에 이미 5건 예약이 있으니 이 예약은 RES-20260320-006"처럼 번호를 부여하는 것입니다.

#### 조회 메서드 — ReservationInfoDto 반환으로 수정

```java
public Optional<ReservationInfoDto> findByReservationNumber(String reservationNumber) {
    return reservationRepository.findByReservationNumber(reservationNumber)
            .map(ReservationInfoDto::new);
}

public List<ReservationInfoDto> findByPhoneAndName(String phone, String name) {
    String trimmedName = name.trim();
    String normalizedPhone = phone.replaceAll("[^0-9]", "");
    return reservationRepository.findByNormalizedPhoneAndName(normalizedPhone, trimmedName)
            .stream().map(ReservationInfoDto::new).toList();
}
```

> **💡 입문자 설명**
> - **이 코드가 하는 일**: 예약번호로 단건 조회하거나, 이름+전화번호로 목록을 조회하여 `ReservationInfoDto`로 변환해 반환합니다. 전화번호는 숫자만 남도록 정규화합니다.
> - **왜 이렇게 썼는지**: `replaceAll("[^0-9]", "")`은 정규식으로 숫자가 아닌 문자(하이픈 등)를 제거합니다. 사용자가 "010-1234-5678"처럼 입력해도 "01012345678"로 정규화하여 DB의 저장 형식과 맞춥니다. `.map(ReservationInfoDto::new)`는 조회된 엔티티를 DTO로 변환합니다.
> - **쉽게 말하면**: 예약번호 또는 이름+전화번호로 예약을 찾아주는 검색 창구입니다. 전화번호는 형식이 달라도 숫자만 비교하여 찾습니다.

### 5. ReservationController — GET /reservation/lookup 추가

예약번호 단건 조회와 이름+전화번호 목록 조회를 하나의 핸들러에서 처리.

```java
@GetMapping("/lookup")
public String lookupPage(HttpServletRequest request) {
    String reservationNumber = request.getParameter("reservationNumber");
    String name  = request.getParameter("name");
    String phone = request.getParameter("phone");

    if (reservationNumber != null && !reservationNumber.isBlank()) {
        reservationService.findByReservationNumber(reservationNumber)
                .ifPresentOrElse(
                        dto -> request.setAttribute("reservation", dto),
                        () -> request.setAttribute("errorMessage", "예약을 찾을 수 없습니다.")
                );
    } else if (name != null && !name.isBlank() && phone != null && !phone.isBlank()) {
        List<ReservationInfoDto> list = reservationService.findByPhoneAndName(phone, name);
        if (list.isEmpty()) {
            request.setAttribute("errorMessage", "예약을 찾을 수 없습니다.");
        } else {
            request.setAttribute("reservations", list);
        }
    }
    request.setAttribute("pageTitle", "예약 조회");
    return "reservation/reservation-lookup";
}
```

> **💡 입문자 설명**
> - **이 코드가 하는 일**: 예약 조회 페이지를 처리합니다. 예약번호가 있으면 단건 조회, 이름+전화번호가 있으면 목록 조회를 수행하고 결과를 `request`에 담아 화면에 전달합니다.
> - **왜 이렇게 썼는지**: `request.getParameter()`로 GET 요청의 쿼리 파라미터를 읽습니다. `ifPresentOrElse(성공처리, 실패처리)`는 Optional에 값이 있을 때와 없을 때를 한 번에 처리하는 Java 메서드입니다. `request.setAttribute()`로 템플릿에 데이터를 전달합니다.
> - **쉽게 말하면**: 예약 조회 창구에서 고객이 예약번호를 가져오면 그걸로 찾고, 이름과 전화번호를 가져오면 그걸로 찾아서 화면에 표시해주는 담당자입니다.

### 6. reservation-lookup.mustache (신규 생성)

두 가지 조회 탭(예약번호 / 이름+전화번호) + 결과 표시 (단건 / 목록).
`canModify`가 `true`인 예약(`RESERVED` 상태)에만 변경·취소 버튼 표시.

```html
{{#canModify}}
<div class="flex gap-3">
  <button data-href="/reservation/modify?reservationNumber={{reservationNumber}}" class="res-btn ...">
    예약 변경
  </button>
  <button data-href="/reservation/cancel?reservationNumber={{reservationNumber}}" class="res-btn ...">
    예약 취소
  </button>
</div>
{{/canModify}}
```

> **💡 입문자 설명**
> - **이 코드가 하는 일**: `canModify`가 `true`인 예약(RESERVED 상태)에만 "예약 변경"과 "예약 취소" 버튼을 표시합니다. 이미 취소되거나 완료된 예약에는 버튼이 보이지 않습니다.
> - **왜 이렇게 썼는지**: Mustache 템플릿에서 `{{#canModify}}...{{/canModify}}`는 `canModify` 값이 `true`일 때만 내용을 렌더링합니다. 서버에서 DTO에 `canModify` 플래그를 넣어두었기 때문에 템플릿에서 간단히 조건 분기가 가능합니다.
> - **쉽게 말하면**: 변경 가능한 예약에만 "변경/취소" 버튼이 나타나는 조건부 표시 로직입니다.

### 7. reservation-modify.mustache — canModify 조건부 폼 표시

`RESERVED` 상태가 아닌 예약 접근 시 변경 폼 대신 안내 메시지 표시.

```html
{{#reservation}}
{{^canModify}}
<div class="bg-amber-50 border border-amber-200 ...">
  이 예약은 현재 <strong>{{status}}</strong> 상태로 변경이 불가합니다.
</div>
{{/canModify}}
{{#canModify}}
<div class="bg-white ...">
  <!-- 진료과 / 의사 / 날짜 / 시간 변경 폼 -->
</div>
{{/canModify}}
{{/reservation}}
```

> **💡 입문자 설명**
> - **이 코드가 하는 일**: 예약 변경 화면에서 `canModify` 값에 따라 다른 내용을 표시합니다. 변경 불가 상태이면 안내 메시지를, 변경 가능 상태이면 변경 폼을 보여줍니다.
> - **왜 이렇게 썼는지**: Mustache에서 `{{^canModify}}`는 "canModify가 false일 때"를 의미하는 반전 조건(`^`)입니다. `{{#reservation}}`으로 예약 데이터가 있을 때만 전체 블록을 렌더링합니다.
> - **쉽게 말하면**: 변경 불가한 예약이면 "변경할 수 없습니다" 안내문을, 변경 가능한 예약이면 변경 폼을 보여주는 조건부 화면 구성입니다.

```

GET /reservation/modify?reservationNumber=RES-...

  └─ 기존 예약 CANCELLED 처리 + 신규 예약 생성
```
