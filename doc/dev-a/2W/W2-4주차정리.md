# [2주차-4] 내 예약 찾아보기 (조회 기능 구현)

이 문서는 환자가 자신의 이름과 전화번호를 입력했을 때, DB 창고에서 그 환자의 예약 내역을 쏙쏙 골라내어 보여주는 과정을 설명합니다.

---

## 🧭 1. 워크플로우(Workflow): 어떤 계획을 세웠나요?
*   **목적**: 환자가 이름과 전화번호만 넣으면 자신의 예약 목록을 볼 수 있게 하자!
*   **설계**: 
    1. 사용자가 이름과 연락처를 입력하는 조회 화면(`reservation-lookup.mustache`)을 만들자.
    2. 입력한 데이터를 서버로 보내면, 서비스(Service)가 DB에서 해당 정보를 찾도록 하자.
    3. 찾은 결과(예약 리스트)를 화면에 다시 뿌려주자.

---

## 🏗️ 2. 리포트(Report): 실제로 어떻게 구현했나요?
DB에서 데이터를 검색하고 화면에 전달하는 과정을 살펴봅시다.

### 2.1. 창고지기의 검색 능력: ReservationRepository.java
전화번호와 이름을 기준으로 DB에서 데이터를 찾는 기능을 추가합니다.

```java
// JpaRepository에 우리가 원하는 검색 조건을 이름 규칙대로 적어주면 스프링이 알아서 검색 쿼리를 만듭니다!
public interface ReservationRepository extends JpaRepository<Reservation, Long> {

    // "이름(name)과 전화번호(phone)가 일치하는 예약들을 찾아줘!"라는 뜻입니다.
    // findAllBy... 라고 적으면 조건에 맞는 모든 데이터를 리스트(List)로 가져옵니다.
    List<Reservation> findAllByNameAndPhone(String name, String phone);
}
```

### 2.2. 검색 결과를 화면에 보내기: ReservationController.java
사용자가 입력한 이름과 번호를 받아 검색을 요청하고 결과를 화면에 보냅니다.

```java
// 1. @PostMapping: 조회 버튼을 눌렀을 때 실행됩니다.
@PostMapping("/lookup")
public String lookupReservation(String name, String phone, HttpServletRequest request) {
    
    // 2. 서비스 호출: "주방장님, 이 이름과 번호로 예약된 것들 다 찾아주세요!"
    List<ReservationInfoDto> list = reservationService.lookupReservations(name, phone);

    // 3. 장바구니 담기: 찾은 예약 리스트를 "reservations"라는 이름으로 화면에 넘깁니다.
    request.setAttribute("reservations", list);
    
    // 4. 결과 화면을 보여줍니다.
    return "reservation/reservation-list";
}
```

---

## 💡 3. 왜 이 코드를 사용했고, 어떤 기능을 하나요?
*   **`List<Reservation>`**: 예약이 한 개가 아닐 수도 있기 때문에, 여러 개의 데이터를 줄줄이 담을 수 있는 **'리스트(기차 칸)'** 형태의 자료구조를 사용합니다.
*   **Query Method (Repository)**: 복잡한 SQL 문장을 직접 짜지 않아도, 메서드 이름(`findAllByNameAndPhone`)만 잘 지으면 스프링이 **자동으로 검색 기능을 구현**해 줍니다.
*   **검색 필터**: 수만 개의 데이터 중에서 딱 **내가 원하는 데이터만 골라내는 핵심 로직**입니다.
