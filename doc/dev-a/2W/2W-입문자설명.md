# 2주차 전체 정리 — Java 입문자를 위한 설명

> 1주차에서 URL → 화면 연결을 배웠다면, 2주차는 **"화면에서 데이터를 입력하고 저장하기"** 전체 흐름을 완성합니다.

---

## 2주차 전체 흐름 요약

```
W2-1: 예약 폼 화면 구성 (form, name 속성, CSRF)
W2-2: 진료과 → 의사 동적 연동 (AJAX, REST API)
W2-3: 날짜 달력에서 의사 근무일만 선택 (Flatpickr)
W2-4: 예약 정보 DB 저장 + 완료 화면 이동 (JPA, PRG 패턴)
W2-5: 예약 조회 · 취소 · 변경 기능 (Service, DTO)
W2-8: 서버 측 입력값 검증 (Bean Validation)
W2-9: 단위 테스트 작성 (JUnit, Mockito)
```

## W2-1: 예약 폼 화면 — form, POST, CSRF

### 배운 개념

**HTML form 과 Spring 연결 방법:**

```html
<!-- Mustache 템플릿 -->
<form method="post" action="/reservation/create">
  <input type="hidden" name="{{_csrf.parameterName}}" value="{{_csrf.token}}">
  <input type="text" name="name" placeholder="이름">
  <input type="tel" name="phone" placeholder="전화번호">
  <button type="submit">예약하기</button>
</form>
```

- `name="name"`: Spring이 이 값을 받아서 파라미터로 매핑할 때 필요한 이름표
- `_csrf.token`: **CSRF 공격 방어**용 숨겨진 토큰. 악성 사이트에서 사용자 몰래 폼을 제출하는 것을 막음

**컨트롤러에서 POST 요청 받기:**

```java
@PostMapping("/reservation/create")
public String create(HttpServletRequest request) {
    String name = request.getParameter("name");
    String phone = request.getParameter("phone");
    // ... 처리
}
```

> **왜 `HttpServletRequest`를 쓰나?**
> 이 프로젝트 규칙: POST 컨트롤러는 `HttpServletRequest`로 파라미터를 받는다.

## W2-2: 진료과 → 의사 동적 연동 — AJAX, REST API

### 배운 개념

**AJAX**: 페이지 새로고침 없이 서버에서 데이터를 가져오는 기술

```javascript
// 진료과를 선택하면 해당 진료과 의사 목록을 서버에서 가져옴
departmentSelect.addEventListener('change', async function() {
    const res = await fetch('/api/reservation/doctors?departmentId=' + this.value);
    const doctors = await res.json();
    // 가져온 의사 목록을 드롭다운에 추가
});
```

**REST API 컨트롤러:**

```java
@RestController  // JSON을 반환하는 컨트롤러
@RequestMapping("/api/reservation")
public class ReservationApiController {

    @GetMapping("/doctors")
    public List<DoctorDto> getDoctors(@RequestParam Long departmentId) {
        return doctorRepository.findByDepartmentId(departmentId)
                               .stream()
                               .map(DoctorDto::new)
                               .collect(Collectors.toList());
    }
}
```

- `@RestController`: HTML이 아닌 JSON 데이터를 반환
- `@RequestParam`: URL의 `?departmentId=1` 같은 쿼리 파라미터를 받음

**겪은 문제 — LazyInitializationException:**
```
LazyInitializationException: could not initialize proxy - no Session
```
JPA에서 연관 객체(`doctor.getDepartment()`)를 가져오려면 DB 연결(세션)이 열려 있어야 하는데,
이미 닫힌 후 접근하면 이 오류가 발생한다.
→ **해결**: DTO로 변환할 때 필요한 값만 미리 추출

## W2-3: 날짜 달력 — Flatpickr, 의사 근무일 제한

### 배운 개념

**Flatpickr**: 달력 UI 라이브러리. 특정 날짜만 선택 가능하도록 설정할 수 있음.

```javascript
const DAY_MAP = { SUN:0, MON:1, TUE:2, WED:3, THU:4, FRI:5, SAT:6 };

// 의사가 MON, WED, FRI만 근무한다면 그 날만 선택 가능하게 설정
const datePicker = flatpickr('#date', {
    minDate: 'today',
    disable: [
        function(date) {
            return !availableDayNums.includes(date.getDay());
        }
    ]
});
```

**겪은 문제 — CSP(Content Security Policy):**
CDN으로 Flatpickr를 불러오면 브라우저가 보안 정책으로 차단.
→ **해결**: Flatpickr JS/CSS 파일을 직접 프로젝트 안(`/static/js`, `/static/css`)에 넣어서 서빙

## W2-4: DB 저장 — JPA, Repository, PRG 패턴

### 배운 개념

**JPA Repository**: DB 저장/조회를 코드로 처리하는 인터페이스

```java
// 인터페이스만 선언하면 Spring이 자동으로 구현체를 만들어줌
public interface ReservationRepository extends JpaRepository<Reservation, Long> {
    Optional<Reservation> findByReservationNumber(String number);
}
```

**Service 계층**: 비즈니스 로직(규칙)을 담당

```java
@Service
@Transactional
public class ReservationService {

    public void createReservation(ReservationCreateForm form) {
        // 1. 환자가 이미 있으면 기존 정보 사용, 없으면 새로 생성
        Patient patient = patientRepository.findByPhone(form.getPhone())
                .orElseGet(() -> patientRepository.save(new Patient(...)));

        // 2. 중복 예약 확인
        if (reservationRepository.existsByDoctorAndDateAndTime(...)) {
            throw new CustomException("이미 예약된 시간입니다.");
        }

        // 3. 예약 저장
        reservationRepository.save(new Reservation(...));
    }
}
```

**PRG 패턴 (Post-Redirect-Get)**:

```
[나쁜 방식] POST /create → 완료 페이지 직접 반환
  → 새로고침 시 폼이 다시 제출되어 중복 저장 발생!

[올바른 방식] POST /create → redirect:/reservation/complete
  → 새로고침해도 GET 요청만 발생, 중복 저장 없음
```

```java
@PostMapping("/create")
public String create(HttpServletRequest request, RedirectAttributes ra) {
    reservationService.createReservation(form);
    ra.addFlashAttribute("message", "예약이 완료되었습니다.");
    return "redirect:/reservation/complete";  // PRG 패턴
}
```

## W2-5: 예약 조회·취소·변경 — DTO, 예약번호

### 배운 개념

**DTO (Data Transfer Object)**: 화면에 보내줄 데이터만 골라서 담는 그릇

```java
public class ReservationInfoDto {
    private String reservationNumber;  // 예약번호
    private String statusText;         // "예약완료", "접수됨" 등 한글 상태
    private boolean canModify;         // 변경 버튼 표시 여부

    public ReservationInfoDto(Reservation r) {
        this.reservationNumber = r.getReservationNumber();
        this.statusText = toKorean(r.getStatus());
        this.canModify = r.getStatus() == ReservationStatus.RESERVED;
    }

    private String toKorean(ReservationStatus status) {
        return switch (status) {
            case RESERVED -> "예약완료";
            case CANCELLED -> "취소됨";
            // ...
        };
    }
}
```

**예약번호 형식**: `RES-20260309-001`
→ 날짜 + 당일 누적 순번으로 생성 (`ReservationNumberGenerator`)

## W2-8: 서버 측 입력값 검증 — Bean Validation

### 배운 개념

**Bean Validation**: `@Valid`와 어노테이션으로 입력값을 자동으로 검사

```java
public class ReservationCreateForm {
    @NotBlank(message = "이름을 입력해주세요.")
    private String name;

    @NotBlank(message = "전화번호를 입력해주세요.")
    @Pattern(regexp = "^01[0-9]-\\d{3,4}-\\d{4}$", message = "올바른 전화번호 형식이 아닙니다.")
    private String phone;
}
```

```java
@PostMapping("/create")
public String create(@Valid ReservationCreateForm form, BindingResult bindingResult) {
    if (bindingResult.hasErrors()) {
        // 검증 실패 → 폼 화면 다시 표시 (에러 메시지 포함)
        return "reservation/direct-reservation";
    }
    // 검증 통과 → 예약 저장
    reservationService.createReservation(form);
    return "redirect:/reservation/complete";
}
```

- **클라이언트 검증** (JavaScript): 빠르지만 우회 가능
- **서버 검증** (Bean Validation): 반드시 필요한 마지막 방어선

## W2-9: 단위 테스트 — JUnit, Mockito

### 배운 개념

**단위 테스트**: 작은 단위(메서드 하나)가 올바르게 동작하는지 확인하는 자동화된 코드

```java
// Service 테스트 (Mockito로 DB 호출 없이 테스트)
@ExtendWith(MockitoExtension.class)
class ReservationServiceTest {

    @Mock
    ReservationRepository reservationRepository;  // 가짜 Repository

    @InjectMocks
    ReservationService reservationService;         // 테스트 대상

    @Test
    @DisplayName("중복 예약 시 예외 발생")
    void create_duplicate_throws() {
        // given: 이미 해당 시간에 예약이 있다고 가짜 응답 설정
        given(reservationRepository.existsByDoctorAndDateAndTime(...)).willReturn(true);

        // when + then: 예외가 발생하는지 확인
        assertThatThrownBy(() -> reservationService.createReservation(form))
            .isInstanceOf(CustomException.class);
    }
}
```

**테스트 종류:**

| 어노테이션 | 용도 | 속도 |
|-----------|------|------|
| `@ExtendWith(MockitoExtension.class)` | Service 테스트 (가짜 DB) | 빠름 |
| `@DataJpaTest` | Repository 테스트 (실제 DB H2) | 보통 |
| `@WebMvcTest` | Controller 테스트 (HTTP 요청/응답) | 보통 |

**2주차 테스트 결과**: 4개 클래스, 13개 테스트, 전원 통과 ✅
