# [3주차-1] AI 증상 분석 흉내 내기 (자바스크립트 비동기 처리)

이 문서는 사용자가 입력한 증상 텍스트를 읽어와서, 적절한 진료과를 추천해 주는 '가짜 AI' 기능을 만드는 과정을 설명합니다.

---

## 🧭 1. 워크플로우(Workflow): 어떤 계획을 세웠나요?
*   **목적**: 사용자가 "기침이 나요"라고 입력하고 분석 버튼을 누르면, "내과"를 추천해 주자!
*   **설계**: 
    1. 특정 단어(열, 기침 등)가 들어오면 어떤 진료과로 보낼지 '단어장(SYMPTOM_MAP)'을 만들자.
    2. 버튼을 누르면 즉시 결과가 나오는 게 아니라, 진짜 분석하는 것처럼 보이게 1.5초 정도 기다렸다가 결과를 보여주자(비동기 로딩 연출).
    3. 추천된 진료과 정보를 가지고 실제 예약 페이지로 이동하게 하자.

---

## 🏗️ 2. 리포트(Report): 실제로 어떻게 구현했나요?
자바스크립트(JS) 코드로 브라우저에서 일어나는 마법을 살펴봅시다.

### 2.1. AI의 지식 베이스: SYMPTOM_MAP
어떤 단어가 어떤 진료과에 해당하는지 정의한 데이터입니다.

```javascript
// SYMPTOM_MAP: 단어와 진료과를 짝지어둔 '데이터 리스트'입니다.
const SYMPTOM_MAP = [
  // 1. keywords 안에 있는 단어 중 하나라도 포함되면 '내과'를 추천합니다.
  { keywords: ['열', '기침', '감기', '콧물', '목', '인후'], dept: '내과',       doctor: '김민준 원장' },
  // 2. '수술', '뼈' 같은 단어가 있으면 '외과'를 추천합니다.
  { keywords: ['수술', '상처', '외상', '찢김', '골절', '뼈'], dept: '외과',     doctor: '이성호 원장' },
  // ... (다른 진료과들도 동일하게 설정)
];

// 만약 사용자가 입력한 단어가 어디에도 없다면 기본적으로 '내과'를 보여줍니다.
const DEFAULT_RESULT = { dept: '내과', doctor: '김민준 원장' };
```

### 2.2. 증상 분석하기: analyzeSymptom 함수
사용자가 입력한 글자 속에서 키워드를 찾아내는 '눈' 역할을 하는 코드입니다.

```javascript
function analyzeSymptom(text) {
  // 1. 단어장(SYMPTOM_MAP)을 하나씩 꺼내어 살펴봅니다.
  for (const entry of SYMPTOM_MAP) {
    // 2. 사용자가 입력한 글자(text) 안에 단어장의 키워드(kw)가 포함되어 있는지 확인합니다.
    if (entry.keywords.some(kw => text.includes(kw))) {
      // 3. 일치하는 단어를 찾았다면 해당 진료과 정보를 돌려줍니다.
      return { dept: entry.dept, doctor: entry.doctor };
    }
  }
  // 4. 끝까지 못 찾았다면 기본값을 돌려줍니다.
  return DEFAULT_RESULT;
}
```

### 2.3. 진짜 분석하는 것처럼 보이기: 비동기 처리(setTimeout)
버튼을 눌렀을 때 1.5초 동안 "분석 중..."이라는 글자를 보여주는 '연출' 코드입니다.

```javascript
// 분석 버튼을 눌렀을 때 실행되는 코드입니다.
analyzeBtn.addEventListener('click', function() {
    
    // 1. 화면에 "AI가 분석 중입니다..."라는 로딩 메시지를 보여줍니다.
    showLoading(true);

    // 2. 1.5초(1500ms) 뒤에 아래 코드를 실행하라고 예약합니다. (비동기 처리)
    setTimeout(() => {
        // 3. 위에서 만든 analyzeSymptom 함수를 써서 결과를 결정합니다.
        const result = analyzeSymptom(inputText.value);
        
        // 4. 화면의 추천 결과 칸에 결정된 진료과 이름을 써넣습니다.
        document.getElementById('rec-dept').innerText = result.dept;
        
        // 5. 로딩 메시지를 숨기고 진짜 결과를 화면에 띄웁니다!
        showLoading(false);
        showResultArea(true);
    }, 1500); 
});
```

---

## 💡 3. 왜 이 코드를 사용했고, 어떤 기능을 하나요?
*   **비동기 처리 (`setTimeout`)**: 웹 프로그램에서 데이터를 불러오거나 분석할 때 시간이 걸리는 경우, 사용자가 지루하지 않게 **'기다리는 중'임을 알려주는 로직**을 구현하기 위해 사용합니다.
*   **`includes`와 `some`**: 글자 속에 특정 단어가 들어있는지 확인하는 아주 편리한 자바스크립트의 **기본 검색 도구**들입니다.
*   **DOM 조작 (`innerText`)**: 서버에 가지 않고도 **브라우저 화면의 글자를 실시간으로 바꾸는** 웹 프론트엔드의 핵심 기능입니다.
