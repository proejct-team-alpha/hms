# [3주차-4] 추천 결과로 자동 예약 폼 채우기 (페이지 간 데이터 전달)

이 문서는 AI 추천을 받은 사용자가 '예약하기' 버튼을 눌렀을 때, 추천받은 진료과·의사·시간 정보가 다음 페이지의 입력칸에 자동으로 입력되게 만드는 과정을 설명합니다.

---

## 🧭 1. 워크플로우(Workflow): 어떤 계획을 세웠나요?
*   **목적**: 사용자가 직접 선택할 필요 없이, AI가 추천해 준 정보를 예약 폼에 미리 채워주자!
*   **설계**: 
    1. 추천 결과 페이지에서 '예약하기'를 누르면, 정보를 주소창(URL)에 담아서 보내자. (예: `?dept=내과&doctor=의사김민준...`)
    2. 예약 폼 페이지(`direct-reservation`)가 열릴 때, 주소창에 있는 정보를 읽어오자.
    3. 읽어온 정보를 바탕으로 진료과, 의사, 시간 선택창(Select)의 값을 자동으로 바꿔주자.

---

## 🏗️ 2. 리포트(Report): 실제로 어떻게 구현했나요?
두 페이지 사이에서 데이터가 오고 가는 과정을 살펴봅시다.

### 2.1. 데이터 보내기: symptom-reservation.mustache
추천받은 텍스트를 주소(URL) 뒤에 붙여서 다음 페이지로 이동시킵니다.

```javascript
function proceedToDirect() {
  // 1. 화면에 표시된 추천 진료과, 의사, 시간 텍스트를 가져옵니다.
  const dept   = document.getElementById('rec-dept').textContent;
  const doctor = document.getElementById('rec-doctor').textContent;
  const time   = document.getElementById('rec-time').textContent;

  // 2. 주소창 뒤에 ?dept=...&doctor=... 처럼 정보를 붙여서 이동합니다.
  // 이렇게 주소 뒤에 붙는 데이터를 '쿼리 파라미터'라고 부릅니다.
  location.href = `/reservation/direct-reservation?dept=${dept}&doctor=${doctor}&time=${time}`;
}
```

### 2.2. 데이터 받아서 채우기: direct-reservation.mustache
새로 열린 페이지에서 주소창의 정보를 읽어와 입력칸을 자동으로 선택합니다.

```javascript
// 1. 주소창에서 정보를 꺼내오는 도구(URLSearchParams)를 사용합니다.
const urlParams = new URLSearchParams(window.location.search);
const recDept   = urlParams.get('dept');   // "내과"
const recDoctor = urlParams.get('doctor'); // "의사김민준"

// 2. 진료과 이름(내과)을 숫자 ID(1)로 바꿔주는 지도를 사용합니다.
const DEPT_ID_MAP = { '내과': '1', '외과': '2', ... };
const deptId = DEPT_ID_MAP[recDept];

// 3. 진료과 선택창의 값을 자동으로 바꿉니다.
deptSelect.value = deptId;

// 4. 의사 선택창도 자동으로 바꿉니다. (서버에서 의사 목록을 가져온 뒤 실행)
fetch(`/api/doctors?deptId=${deptId}`)
  .then(res => res.json())
  .then(doctors => {
    // 의사 목록 중에서 이름이 일치하는 분을 찾아서 선택합니다.
    const matched = Array.from(doctorSelect.options).find(o => o.textContent === recDoctor);
    if (matched) {
        doctorSelect.value = matched.value;
    }
  });
```

---

## 💡 3. 왜 이 코드를 사용했고, 어떤 기능을 하나요?
*   **URLSearchParams**: 주소창의 복잡한 텍스트에서 **내가 원하는 데이터만 이름으로 쏙 뽑아낼 때** 사용하는 아주 유용한 도구입니다.
*   **`dispatchEvent(new Event('change'))`**: 프로그램이 코드로 값을 바꿨을 때, 브라우저에게 **"값이 바뀌었으니 관련된 다른 기능들도 업데이트해!"**라고 신호를 보내는 역할을 합니다. (예: 의사가 바뀌면 진료 가능한 날짜 달력도 바뀌어야 함)
*   **데이터 매핑 (DEPT_ID_MAP)**: 화면에서 쓰는 '이름'과 서버에서 쓰는 '숫자 ID' 사이를 이어주는 **번역기** 역할을 합니다.
