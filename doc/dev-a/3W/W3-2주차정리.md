# [3주차-2] 비동기 처리의 정석 (async/await 구조 잡기)

이 문서는 나중에 진짜 AI 서버와 통신할 때를 대비해서, 코드를 더 전문적이고 튼튼하게(`async/await`) 바꾸는 과정을 설명합니다.

---

## 🧭 1. 워크플로우(Workflow): 어떤 계획을 세웠나요?
*   **목적**: 단순한 기다리기(`setTimeout`) 대신, 실제 서버와 대화하는 방식(`fetch`와 `async/await`)으로 코드의 뼈대를 바꾸자!
*   **설계**: 
    1. `callSymptomApi`라는 전용 함수를 만들어서 "나중에 여기서 진짜 AI랑 통신할 거야!"라고 자리를 맡아두자.
    2. 버튼 클릭 이벤트를 `async` 함수로 만들어서, 결과가 올 때까지 우아하게 기다리게(`await`) 하자.
    3. 만약 서버가 고장 났을 때를 대비해 에러 처리(`try/catch`)도 미리 준비하자.

---

## 🏗️ 2. 리포트(Report): 실제로 어떻게 구현했나요?
더 현대적이고 깔끔해진 자바스크립트 코드를 살펴봅시다.

### 2.1. 미래를 위한 약속: callSymptomApi 함수
지금은 가짜 데이터를 주지만, 나중에 진짜 AI 서버 주소만 넣으면 바로 작동하게 만든 함수입니다.

```javascript
// 1. async: 이 함수는 시간이 좀 걸리는 '비동기' 작업을 포함하고 있다는 뜻입니다.
async function callSymptomApi(symptomText) {
  // 2. Promise: "내가 지금 당장은 결과가 없지만, 조금 이따가 꼭 가져다줄게!"라는 약속입니다.
  return new Promise(resolve => {
    // 3. 1.5초 뒤에 결과를 가져다주기로 약속(resolve)합니다.
    setTimeout(() => {
        const dummyResult = analyzeSymptom(symptomText);
        resolve(dummyResult);
    }, 1500);
  });
}
```

### 2.2. 기다림의 미학: analyzeBtn 클릭 핸들러
결과가 나올 때까지 다른 일을 방해하지 않고 조용히 기다리는 코드입니다.

```javascript
// 1. async: 클릭 이벤트 함수도 비동기 처리를 할 수 있게 만듭니다.
analyzeBtn.addEventListener('click', async function() {
    
    // 2. try-catch: 혹시라도 서버가 죽거나 에러가 나면 프로그램이 멈추지 않게 방어합니다.
    try {
        showLoading(true);

        // 3. await: "결과가 올 때까지 여기서 잠시 멈춰서 기다려!"라는 뜻입니다.
        // 이 한 줄 덕분에 코드가 위에서 아래로 흐르는 것처럼 아주 읽기 쉬워집니다.
        const result = await callSymptomApi(inputText.value);

        // 4. 결과가 도착하면 화면을 업데이트합니다.
        updateResultUI(result);
        
    } catch (error) {
        // 5. 에러가 나면 사용자에게 알려주고 버튼을 다시 활성화합니다.
        alert("분석 중에 문제가 생겼어요. 다시 시도해 주세요.");
        resetButtonUI();
    } finally {
        showLoading(false);
    }
});
```

---

## 💡 3. 왜 이 코드를 사용했고, 어떤 기능을 하나요?
*   **`async/await`**: 복잡한 비동기 코드(콜백 지옥)를 **마치 일반적인 순서대로 실행되는 코드처럼** 보이게 만들어줍니다. 가독성이 10배는 좋아집니다!
*   **`Promise`**: 미래의 어느 시점에 완료될 작업을 표현하는 객체입니다. 자바스크립트에서 **서버 통신의 핵심**입니다.
*   **`try/catch`**: 프로그램의 **안정성**을 위한 필수 장치입니다. 예상치 못한 상황(인터넷 끊김 등)에서도 서비스가 계속 유지되게 돕습니다.
