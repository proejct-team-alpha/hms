# W3-8 설계 - 물품 담당자 페이지 페이징 기능

**날짜**: 2026-03-18
**대상 역할**: 물품 담당자 (`/item-manager`)

---

## 개요

물품 담당자의 3개 페이지에 클라이언트 사이드 JS 페이징을 추가한다. 백엔드 변경 없음.

---

## 요구사항

- 페이지당 **10건**
- 번호 버튼 최대 5개 + `...` 생략 + 이전/다음 버튼
- 필터/검색 변경 시 **1페이지로 리셋**
- 총 항목 0건 또는 총 페이지 1개이면 **페이지 바 숨김**
- 페이지 상태는 메모리(JS 변수)에만 유지 — 새로고침 시 1페이지로 복귀

---

## 번호 버튼 생성 로직 (공통)

```
totalPages <= 5: 모든 번호 표시
totalPages > 5:
  - 현재 페이지 중심으로 window = [cur-2, cur+2] (경계 조정)
  - window가 마지막 페이지를 포함하면 '...' 및 마지막 페이지 버튼 숨김
  - window가 마지막 페이지에 닿지 않으면 window 끝 + '...' + 마지막 페이지 표시

예시:
page 1/12:  이전  [1][2][3][4][5]...[12]  다음
page 6/12:  이전  [4][5][6][7][8]...[12]  다음
page 10/12: 이전  [8][9][10][11][12]      다음
```

> **💡 입문자 설명**
> - **이 코드가 하는 일**: 번호 버튼을 몇 개까지, 어떻게 표시할지를 정의한 설계 로직입니다. 전체 페이지가 5개 이하면 전부 표시하고, 6개 이상이면 현재 페이지를 중심으로 5개 버튼만 보여주고 나머지는 `...`으로 생략합니다.
> - **왜 이렇게 썼는지**: 페이지가 100개면 번호 버튼 100개를 모두 보여줄 수 없습니다. 현재 위치 주변 5개만 보여주고 마지막 페이지로 바로 뛸 수 있는 버튼을 제공하는 것이 일반적인 UX 패턴입니다.
> - **쉽게 말하면**: 책의 목차처럼, 현재 읽고 있는 페이지 주변 것들만 보여주고 나머지는 `...`으로 생략해 깔끔하게 표시하는 방법입니다.

## 대상 페이지 및 구현 세부사항

### 1. 물품 목록 (`item-manager/item-list.mustache`)

**추가 기능: 검색창** (초성·영문 지원)
- 카테고리 필터 아래, 테이블 위에 검색 입력창 추가
- 검색어 기준: `<td>` 첫 번째 셀(물품명) 텍스트
- 검색 변경 시 → 1페이지 리셋 후 재렌더

**구현**:
- `<tbody>`에 `id="item-list-tbody"` 추가
- 빈 상태 행에 `id="empty-row"` 부여
- JS에서 `#item-list-tbody tr:not(#empty-row)`를 대상으로 검색 필터링 + 페이징

### 2. 물품 출고 (`item-manager/item-use.mustache`)

#### ① 오늘 출고 내역 테이블
- 내부 `<tbody>`에 `id="today-log-tbody"` 추가
- JS에서 `#today-log-tbody tr`를 선택하여 페이징
- `todayLogs`가 비어 있으면 Mustache 조건으로 페이징 바 숨김

#### ② 카드 그리드
- 기존 `applyFilter()` 함수에 페이징 통합
- 페이징은 **필터링된 카드 부분집합에만** 적용
- 카테고리/검색 변경 시 `cardPage = 1` 리셋

### 3. 입출고 내역 (`item-manager/item-history.mustache`)

- `PAGE_SIZE`: 15 → **10**
- 기존 이전/다음 버튼 사이에 번호 버튼 div 추가
- `render()` 함수에 번호 버튼 생성 로직 통합
