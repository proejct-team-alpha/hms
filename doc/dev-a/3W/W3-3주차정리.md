# [3주차-3] 더 풍부한 추천 결과 제공 (추천 시간 항목 추가)

이 문서는 AI 추천 결과 화면에 '진료 시간'이라는 항목을 새로 추가하여, 사용자에게 더 구체적인 예약 정보를 제안하는 과정을 설명합니다.

---

## 🧭 1. 워크플로우(Workflow): 어떤 계획을 세웠나요?
*   **목적**: 진료과와 의사만 알려주던 것에서 한 걸음 더 나아가, "몇 시에 진료받으시면 좋아요"라고 시간까지 추천해 주자!
*   **설계**: 
    1. 화면(HTML)에 추천 시간을 표시할 수 있는 세 번째 칸(`rec-time`)을 만들자.
    2. 단어장(SYMPTOM_MAP) 데이터에 각 진료과별 추천 시간 정보를 추가하자.
    3. 자바스크립트가 분석 결과를 가져올 때 시간 정보도 함께 가져와서 화면에 그려주자.

---

## 🏗️ 2. 리포트(Report): 실제로 어떻게 구현했나요?
추천 항목이 2개에서 3개로 늘어나는 과정을 살펴봅시다.

### 2.1. 화면에 새 자리 만들기: HTML 수정
기존에 2칸이었던 결과창을 3칸(Grid 3열)으로 넓히고 시간 정보를 담을 자리를 만들었습니다.

```html
<!-- grid-cols-3: 한 줄에 3개의 카드를 나란히 배치하라는 뜻입니다. -->
<div class="grid grid-cols-3 gap-4 mb-6">
  <!-- 1. 추천 진료과 카드 -->
  <div id="rec-dept">...</div>
  
  <!-- 2. 추천 전문의 카드 -->
  <div id="rec-doctor">...</div>
  
  <!-- 3. [신규!] 추천 시간 카드 -->
  <!-- id="rec-time" 이라는 이름을 붙여서 자바스크립트가 찾기 쉽게 했습니다. -->
  <div id="rec-time">여기에 추천 시간이 나옵니다</div>
</div>
```

### 2.2. 데이터 보강하기: SYMPTOM_MAP 업데이트
단어장에 `time`이라는 정보를 새로 추가했습니다.

```javascript
const SYMPTOM_MAP = [
  // 각 진료과마다 가장 빠른 진료 가능 시간을 더미 데이터로 넣어두었습니다.
  { keywords: ['열', '기침'], dept: '내과', doctor: '김민준 원장', time: '09:00' },
  { keywords: ['수술', '뼈'], dept: '외과', doctor: '이성호 원장', time: '10:00' },
  // ...
];
```

### 2.3. 화면에 시간 뿌려주기: JS 업데이트
분석이 끝나면 새롭게 추가된 시간 카드에도 글자를 써넣습니다.

```javascript
// 분석 결과(result)에서 시간 정보를 꺼내어 id가 'rec-time'인 요소에 넣어줍니다.
document.getElementById('rec-time').textContent = result.time;
```

---

## 💡 3. 왜 이 코드를 사용했고, 어떤 기능을 하나요?
*   **Grid Layout (`grid-cols-3`)**: 복잡한 화면 구성을 **격자(Grid) 형태**로 아주 쉽게 정렬해 주는 CSS 기법입니다.
*   **데이터 확장성**: 단어장에 `time`이라는 필드를 추가한 것처럼, 나중에 '진료비'나 '준비물' 같은 정보를 추가하고 싶을 때도 **데이터 구조만 확장**하면 쉽게 구현할 수 있습니다.
*   **UI/UX 개선**: 사용자에게 더 많은 유용한 정보(시간)를 미리 제공함으로써, **예약 결정에 도움**을 주는 기능을 수행합니다.
