# [3주차-5] 분석 실패 시 대처하기 (안내 메시지와 자동 이동)

이 문서는 AI 분석 중에 문제가 생겼을 때, 사용자에게 친절하게 상황을 알리고(Toast 알림) 직접 선택 화면으로 자동으로 안내하는 과정을 설명합니다.

---

## 🧭 1. 워크플로우(Workflow): 어떤 계획을 세웠나요?
*   **목적**: 서버 점검이나 인터넷 문제로 AI 분석이 안 될 때, 사용자가 멍하니 기다리지 않게 하자!
*   **설계**: 
    1. 화면 아래쪽에 살며시 나타났다 사라지는 '에러 알림창(Toast)'을 미리 숨겨두자.
    2. 자바스크립트의 `catch` 블록(에러 발생 시 실행되는 곳)에서 이 알림창을 보여주자.
    3. 3초 뒤에 자동으로 직접 예약 페이지(`direct-reservation`)로 보내주자.

---

## 🏗️ 2. 리포트(Report): 실제로 어떻게 구현했나요?
에러 상황을 우아하게 처리하는 코드를 살펴봅시다.

### 2.1. 숨겨진 안내장: HTML Toast 알림
평소에는 `hidden` 클래스로 숨겨져 있다가, 문제가 생길 때만 나타나는 안내창입니다.

```html
<!-- id="error-toast": 자바스크립트가 부를 때 쓰는 이름입니다. -->
<!-- hidden: 처음에는 보이지 않게 숨겨둡니다. -->
<div id="error-toast" class="hidden fixed bottom-6 ...">
  <i data-feather="alert-circle"></i>
  <span>AI 분석에 실패했습니다. 직접 선택 화면으로 이동합니다.</span>
</div>
```

### 2.2. 에러 발생 시 출동!: JS catch 블록
프로그램에 문제가 생겼을 때만 실행되는 '비상대책' 코드입니다.

```javascript
try {
  // 정상적인 분석 시도...
  const result = await callSymptomApi(text);
  updateUI(result);

} catch (error) {
  // [비상!] 분석 중에 에러가 났다면?
  
  // 1. 숨겨두었던 안내창(Toast)에서 'hidden' 딱지를 떼서 화면에 보여줍니다.
  errorToast.classList.remove('hidden');

  // 2. 3초(3000ms)만 기다렸다가...
  setTimeout(() => {
    // 3. 직접 예약 페이지로 강제로 이동시킵니다. "분석은 안 됐지만 예약은 도와드릴게요!"
    location.href = '/reservation/direct-reservation';
  }, 3000);
}
```

---

## 💡 3. 왜 이 코드를 사용했고, 어떤 기능을 하나요?
*   **Toast 알림**: 화면 전체를 가리는 무거운 팝업 대신, 아래쪽에 살짝 띄워서 **사용자 흐름을 방해하지 않으면서도 정보를 전달**할 때 사용합니다.
*   **`setTimeout` (지연 이동)**: 알림창을 띄우자마자 페이지를 옮기면 사용자가 메시지를 읽을 시간이 없습니다. **사용자가 상황을 이해할 시간(3초)**을 주기 위해 사용합니다.
*   **안정성 (Reliability)**: 시스템의 일부(AI)가 고장 나더라도 전체 서비스(예약)가 중단되지 않게 하는 **서비스의 연속성**을 확보합니다.
