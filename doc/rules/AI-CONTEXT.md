# doc/rules

## 목적

애플리케이션 개발 시 준수해야 할 모든 규칙(Spring, CSS, JavaScript, Test 등)을 명시한다.

## 주요 파일

| 파일명 | 설명 |
|--------|------|
| rule_spring.md | Spring Boot/Java 관련 코딩 컨벤션 (DTO, Service, Controller 패턴 등) |
| rule_css.md | Tailwind CSS 및 BEM 사용 규칙, CSS 구조 가이드 |
| rule_javascript.md | JavaScript 코드 스타일, async/await 사용, 네이밍 규칙 |
| rule_test.md | 테스트 코드 작성 규칙 및 단위/통합 테스트 가이드 |
| rule-controller.md | 컨트롤러 구현 시 PRG 패턴, 응답 객체 사용 세부 규칙 |
| rule-repository.md | 레포지토리 명명 규칙 및 Querydsl 사용 가이드 |

## AI 작업 지침

- 모든 신규 코드 작성 및 리팩토링 시 이 디렉토리의 규칙을 최우선으로 준수한다.
- `AI-CONTEXT.md` 파일 업데이트 시에도 각 기술별 규칙과 일관성을 유지한다.
