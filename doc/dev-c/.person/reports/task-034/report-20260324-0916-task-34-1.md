## 4. 기술 딥다이브 (Technical Deep-dive)
- **관리자 환자 그래프 경로**: AdminDashboardController는 SSR 통계 카드만 내려주고, 실제 그래프 데이터는 AdminDashboardApiController -> AdminDashboardStatsService.getDashboardChart()에서 받아온다. 프론트는 dmin-dashboard.js가 daily-patient-canvas에 custom canvas로 직접 그린다.
- **환자 그래프 변경 포인트**: 데이터는 AdminDashboardChartResponse.dailyPatients로 이미 안정적이다. 따라서 이 그래프는 DTO나 서비스보다 dmin/dashboard.mustache와 dmin-dashboard.js의 시각 요소 수정이 중심이 된다.
- **현재 관리자 물품 그래프 경로**: 같은 AdminDashboardChartResponse 안의 categoryCounts를 받아 dmin-dashboard.js의 enderCategoryChart()에서 가로 막대 그래프로 그리고 있다.
- **기준이 되는 물품 그래프 경로**: 물품 담당자 대시보드는 ItemManagerController -> ItemManagerService.getDashboard() -> ItemDashboardDto.chartDays 경로를 사용한다. uildChartDays()가 최근 7일의 IN/OUT 합계와 높이 비율까지 계산하고, 템플릿은 이를 SSR 막대 그래프로 렌더링한다.
- **다음 단계 영향 범위**: 물품 그래프를 바꾸려면 AdminDashboardChartResponse 계약 자체가 바뀌어야 하므로, AdminDashboardApiControllerTest와 AdminDashboardStatsServiceTest도 함께 수정해야 한다. 환자 그래프는 반대로 API 계약은 유지하고 스타일만 조정하면 된다.
