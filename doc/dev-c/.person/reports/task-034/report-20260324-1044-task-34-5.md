- **작업 일시**: 2026-03-24 10:44
- **진행 상태**: 완료

## 1. 전체 작업 흐름 (Workflow)
1. 관리자 대시보드 물품 그래프가 데이터는 최근 7일 입고/출고 구조로 바뀌었지만, 시각적으로는 아직 물품 담당자 대시보드 느낌과 거리가 있다는 점을 먼저 정리했다.
2. 물품 담당자 대시보드 그래프가 canvas 차트보다 `점선 가이드 + 날짜별 쌍막대 + 하단 라벨` 구조라는 점을 기준으로 삼았다.
3. 관리자 대시보드 템플릿에서 물품 그래프 영역을 `item-flow-chart` DOM 컨테이너로 바꾸고, 그 안에 JS가 막대를 채워 넣는 방향으로 전환했다.
4. 관리자 프런트 스크립트는 `itemFlowDays` 데이터를 받아 입고/출고 쌍막대와 점선 배경을 가진 마크업을 조립하도록 바꿨다.
5. 카드 제목, 범례, 보조 문구, 막대 간격을 물품 담당자 대시보드에 더 가깝게 맞추면서도 관리자 대시보드 카드 레이아웃 안에서는 어색하지 않게 정리했다.
6. 마지막으로 관리자 대시보드 범위 테스트를 다시 실행해 회귀가 없는지 확인했다.

## 2. 핵심 전달 코드 (Core Logic)
```javascript
function renderItemFlowChart(itemFlowDays) {
  const container = document.getElementById('item-flow-chart');
  if (!container) {
    return;
  }

  container.innerHTML = `
    <div class="h-[220px] w-full flex items-end justify-between gap-2 pb-6 border-b border-slate-100 relative">
      <div class="absolute inset-0 flex flex-col justify-between pointer-events-none pb-6">
        <div class="w-full border-t border-dashed border-slate-200 h-0"></div>
        <div class="w-full border-t border-dashed border-slate-200 h-0"></div>
        <div class="w-full border-t border-dashed border-slate-200 h-0"></div>
        <div class="w-full border-t border-dashed border-slate-200 h-0"></div>
      </div>
      ${itemFlowDays.map((item) => renderItemFlowColumn(item)).join('')}
    </div>
  `;
}
```

## 4. 기술 딥다이브 (Technical Deep-dive)
- **렌더링 방식 재선택**: Task 34-4에서는 빠르게 연결하기 위해 Chart.js 막대 차트로 붙였지만, 실제 기준 화면은 SSR 막대에 더 가깝다. 이번 단계에서는 물품 그래프만큼은 DOM 기반 렌더링이 기준 화면과 더 맞다고 판단해 `item-flow-chart` 컨테이너에 막대 마크업을 주입하는 방식으로 바꿨다.
- **시각 기준 통일**: 점선 가이드 라인, 입고/출고 쌍막대, 하단 날짜 라벨, 상단 범례 순서를 물품 담당자 대시보드와 같은 방향으로 맞췄다. 색상은 입고 `indigo`, 출고 `slate`를 유지해 기존 디자인 톤도 이어지게 했다.
- **데이터 재사용**: 렌더링 방식은 바뀌었지만 데이터는 계속 `itemFlowDays`를 사용한다. 즉 Task 34-4에서 바꾼 최근 7일 입고/출고 집계 계약은 그대로 재사용되고, 이 단계는 순수하게 시각 표현만 바꾼 작업이다.
- **에러/빈 상태 처리**: 컨테이너 기반 렌더링으로 바뀌면서 `item-flow-canvas` 클리어 대신 `clearItemFlowChart()`로 HTML을 비우는 방식으로 정리했다. 빈 데이터일 때는 카드 안에서 `데이터 없음` 문구를 직접 렌더링해 사용자 입장에서 어색하지 않게 맞췄다.
- **범위 안전성**: 환자 그래프는 Chart.js line chart를 유지하고, 물품 그래프만 DOM 막대로 가져가 두 그래프의 역할과 시각 언어를 분리했다. 관리자 대시보드 범위 테스트만 다시 돌려 현재 단계의 회귀가 없는지 확인했다.
