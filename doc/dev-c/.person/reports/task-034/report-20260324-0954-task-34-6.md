## 2. 핵심 전달 코드 (Core Logic)

```java
mockMvc.perform(get("/admin/dashboard/chart").with(user("admin").roles("ADMIN")))
        .andExpect(status().isOk())
        .andExpect(jsonPath("$.body.itemFlowDays[0].inHeight").value(80))
        .andExpect(jsonPath("$.body.itemFlowDays[0].outHeight").value(40))
        // 기존 카테고리 재고 계약은 더 이상 내려오지 않는다는 점을 같이 고정한다.
        .andExpect(jsonPath("$.body.categoryCounts").doesNotExist());
```

- 위 검증으로 관리자 대시보드 물품 그래프가 이제 `categoryCounts`가 아니라 `itemFlowDays` 계약을 사용한다는 점을 테스트 레벨에서 고정했다.
- 같은 단계에서 `AdminDashboardControllerTest`는 `daily-patient-canvas`, `item-flow-chart` DOM id와 그래프 카드 제목을 확인하도록 보강했다.
- `AdminDashboardStatsServiceTest`는 로그가 하나도 없을 때도 최근 7일이 0값으로 채워지는지 확인해, 프런트가 빈 배열 대신 안정적인 7일 구조를 받도록 막아뒀다.

## 4. 기술 딥다이브 (Technical Deep-dive)

- **SSR 렌더링 테스트**: 관리자 대시보드는 Mustache SSR과 JS 차트가 섞여 있어서, JSON만 맞아도 화면이 깨질 수 있다. 그래서 `AdminDashboardControllerTest`에서 카드 제목, 캔버스 id, DOM 컨테이너 id를 직접 확인해 화면 골격 회귀를 막았다.
- **API 계약 테스트**: 관리자 물품 그래프는 이번 리팩토링으로 `categoryCounts`에서 `itemFlowDays`로 계약이 바뀌었다. `AdminDashboardApiControllerTest`에서 높이 필드와 구계약 제거를 함께 검증해, 프런트와 백엔드가 다른 DTO를 바라보는 문제를 예방했다.
- **제로필 데이터 보장**: 최근 7일 그래프는 데이터가 없는 날도 일자 축이 유지돼야 화면이 안정적이다. `AdminDashboardStatsServiceTest`에 무로그 시나리오를 추가해 7일 `0값` 구조가 항상 내려오도록 검증했다.
