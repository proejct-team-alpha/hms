- **작업 일시**: 2026-03-24 09:58

## 1. 전체 작업 흐름 (Workflow)
1. 관리자 대시보드 환자 그래프가 현재 `dailyPatients` 계약으로 안정적으로 데이터를 받고 있다는 점을 다시 확인했다.
2. 원무과 대시보드 그래프의 시각적 기준을 그대로 가져오되, 관리자 그래프의 데이터 의미는 건드리지 않는 쪽으로 구현 범위를 좁혔다.
3. 관리자 대시보드 템플릿에서 환자 그래프 카드의 제목, 배지, 설명 문구, 그래프 컨테이너 레이아웃을 원무과 카드 톤에 맞게 재배치했다.
4. 기존 custom canvas 그래프 구현을 Chart.js line chart로 바꿔 축, 격자선, 툴팁, 영역 채우기 톤이 원무과 그래프와 비슷하게 보이도록 정리했다.
5. 물품 그래프는 이번 단계에서 손대지 않고 그대로 두어, 다음 Task 34-4의 데이터 계약 변경과 충돌하지 않게 분리했다.
6. 마지막으로 관리자 대시보드 범위 테스트를 다시 실행해 회귀가 없는지 확인했다.

```javascript
function renderDailyPatientChart(dailyPatients) {
  const labels = dailyPatients.map((item) => formatDateLabel(item.date));
  const values = dailyPatients.map((item) => Number(item.patientCount) || 0);

  destroyDailyPatientChart();

  dailyPatientChart = new Chart(canvas.getContext('2d'), {
    type: 'line',
    data: {
      labels,
      datasets: [
        {
          label: '환자 수',
          data: values,
          backgroundColor: 'rgba(99, 102, 241, 0.08)',
          borderColor: '#6366f1',
          borderWidth: 3,
          fill: true,
          tension: 0.32
        }
      ]
    },
    options: {
      plugins: {
        legend: { display: false }
      },
      scales: {
        y: {
          beginAtZero: true,
          grid: { color: '#f1f5f9' }
        },
        x: {
          grid: { display: false }
        }
      }
    }
  });
}
```

## 4. 기술 딥다이브 (Technical Deep-dive)
- **데이터 계약 유지**: 가장 중요한 기준은 `AdminDashboardChartResponse.dailyPatients`를 그대로 유지하는 것이었다. 이 계약을 건드리지 않으면 `AdminDashboardStatsService`의 최근 7일 집계와 zero-fill 로직을 손대지 않아도 된다.
- **렌더링 방식 전환**: 기존에는 custom canvas로 직접 막대를 그렸는데, 이번에는 Chart.js line chart를 사용했다. 이유는 원무과 그래프와 같은 축/격자선/툴팁/영역 채우기 톤을 빠르고 안정적으로 맞추기 쉽기 때문이다.
- **스타일 기준 적용**: 인디고 강조색, 옅은 슬레이트 격자선, 작은 축 라벨, 어두운 툴팁, 부드러운 영역 채우기, hover point, 여백이 안정적인 카드 레이아웃을 적용했다. 범례는 단일 데이터셋이므로 숨겨서 시각적 밀도를 맞췄다.
- **범위 분리**: 이번 단계는 환자 그래프만 바꿨다. 관리자 물품 그래프는 `categoryCounts` 기반 구조를 그대로 유지해, 다음 단계에서 DTO/API 계약을 바꾸는 작업과 분리했다.
- **리스크 관리**: Chart.js를 관리자 대시보드에 새로 로드하므로 라이브러리 부재 상황에서 깨지지 않게 `typeof Chart` 가드를 넣고, 빈 데이터일 때는 기존과 같은 빈 캔버스 문구를 그리도록 유지했다.
