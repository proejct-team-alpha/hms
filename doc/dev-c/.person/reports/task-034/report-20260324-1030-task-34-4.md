## 1. 전체 작업 흐름 (Workflow)
1. 관리자 대시보드 물품 그래프가 아직 `categoryCounts` 기반의 카테고리별 재고 스냅샷이라는 점을 다시 확인했다.
2. 물품 담당자 대시보드의 최근 7일 그래프가 `ItemDashboardDto.chartDays`를 통해 최근 7일 입고/출고 흐름을 서버에서 계산한다는 기준을 다시 잡았다.
3. 관리자 대시보드 DTO를 `itemFlowDays + dailyPatients` 구조로 바꿔 물품 그래프가 최근 7일 흐름 데이터를 받을 수 있게 했다.
4. `AdminDashboardStatsService`가 `ItemStockLogRepository`를 사용해 최근 7일 입고/출고 합계와 높이 퍼센트를 직접 계산하도록 정리했다.
5. 관리자 대시보드 프런트는 새 `itemFlowDays` 계약을 읽어 입고/출고 2시리즈 막대 그래프로 렌더링하도록 바꿨다.
6. 마지막으로 API/서비스 테스트를 새 계약 기준으로 갱신하고 대시보드 범위 테스트를 다시 실행했다.

## 2. 핵심 전달 코드 (Core Logic)
```java
public record AdminDashboardChartResponse(
        List<ItemFlowDay> itemFlowDays,
        List<DailyPatientCount> dailyPatients) {

    public record ItemFlowDay(
            String label,
            int inAmount,
            int outAmount,
            int inHeight,
            int outHeight) {
    }
}

private List<AdminDashboardChartResponse.ItemFlowDay> buildItemFlowDays(LocalDate today) {
    LocalDateTime start = today.minusDays(6).atStartOfDay();
    LocalDateTime endExclusive = today.plusDays(1).atStartOfDay();
    List<ItemStockLog> logs = itemStockLogRepository.findByCreatedAtBetweenOrderByCreatedAtAsc(start, endExclusive);

    Map<LocalDate, int[]> dailyFlowMap = new LinkedHashMap<>();
    for (int i = 6; i >= 0; i--) {
        dailyFlowMap.put(today.minusDays(i), new int[]{0, 0});
    }

    for (ItemStockLog log : logs) {
        int[] amounts = dailyFlowMap.get(log.getCreatedAt().toLocalDate());
        if (log.getType() == ItemStockType.IN) amounts[0] += log.getAmount();
        else amounts[1] += log.getAmount();
    }

    return dailyFlowMap.entrySet().stream()
            .map(entry -> toItemFlowDay(entry, maxAmount))
            .toList();
}
```

## 4. 기술 딥다이브 (Technical Deep-dive)
- **DTO 계약 교체**: `AdminDashboardChartResponse`에서 `categoryCounts`를 제거하고 `itemFlowDays`를 추가했다. 이제 관리자 차트 API는 물품 그래프용으로 카테고리 합계 대신 최근 7일 입고/출고 흐름을 반환한다.
- **집계 기준 통일**: `AdminDashboardStatsService`는 `ItemStockLogRepository`를 직접 사용해 최근 7일 고정, 날짜별 IN/OUT 합산, 최대값 기준 높이 퍼센트 계산을 수행한다. 이 계산식은 물품 담당자 대시보드의 `buildChartDays()`와 같은 기준을 따른다.
- **프런트 연결 변경**: `admin-dashboard.js`는 더 이상 `categoryCounts`를 읽지 않고 `itemFlowDays`를 읽어 `입고`/`출고` 2개 데이터셋을 가진 Chart.js bar chart를 렌더링한다. 이 단계에서 프런트가 새 API 계약을 소비할 수 있게 연결을 닫았다.
- **테스트 갱신**: `AdminDashboardApiControllerTest`는 새 JSON 경로 `itemFlowDays[*]`를 검증하고, `AdminDashboardStatsServiceTest`는 실제 입고/출고 로그를 입력해 최근 7일 데이터와 높이 퍼센트가 올바르게 계산되는지 확인하도록 바꿨다.
- **범위 분리 유지**: 환자 그래프는 이번 단계에서 건드리지 않았다. 물품 그래프 계약 변경만 먼저 닫고, 다음 단계에서 카드 톤과 시각 디테일을 더 물품 담당자 화면에 맞추는 흐름이 유지된다.
