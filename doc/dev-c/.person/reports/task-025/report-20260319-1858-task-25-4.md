## 1. 작업 목표
`GET /admin/rule/list` 백엔드 응답을 실제 관리자 검색형 목록 화면으로 연결한다.
이번 단계에서는 `rule-list.mustache`에 제목 검색, 카테고리 필터, 활성 여부 필터, 상태 배지, 빈 결과 메시지, 페이지네이션 UI를 붙여서 규칙 목록 화면을 완성하는 것이 목표였다.

## 3. 핵심 구현 내용
### 3-1. 검색/필터 바 UI 추가
- `rule-list.mustache` 상단에 제목 검색 입력창을 추가했다.
- 카테고리 셀렉트와 활성 여부 셀렉트를 추가했다.
- 조회 버튼과 초기화 버튼을 함께 배치해 관리자 목록 화면 패턴에 맞췄다.

### 3-2. 검색 조건 유지 구조 추가
- Mustache에서 `selected`를 자연스럽게 처리하려고 `AdminRuleFilterOptionResponse`를 추가했다.
- `AdminRuleListResponse`에 `categoryOptions`, `activeOptions`를 넣어 화면에서 현재 선택 상태를 바로 렌더링할 수 있게 했다.

### 3-3. 카드 목록 정리
- 기존 카드형 목록 구조는 유지했다.
- 대신 각 카드에 `categoryText`, `activeText`, `activeBadgeClass`를 붙여서 관리용 상태 확인이 쉬워지도록 정리했다.
- 빈 결과일 때는 `조회된 규칙이 없습니다.` 메시지를 별도 카드로 보여주도록 했다.

### 3-4. 페이지네이션 UI 반영
- 이전/다음 링크와 페이지 번호 링크를 하단에 추가했다.
- 링크는 현재 필터/검색 조건을 유지하도록 서비스가 만든 URL을 그대로 사용한다.

### 3-5. 인코딩/BOM 정리
- PowerShell 기본 `utf8` 저장이 BOM을 붙이면서 자바 컴파일이 깨지는 문제가 한 번 있었다.
- 수정한 자바/템플릿/테스트 파일을 UTF-8 without BOM으로 다시 저장해 문제를 해결했다.

## 4. 테스트 및 검증
- 실행 명령: `./gradlew test --tests 'com.smartclinic.hms.admin.rule.*'`
- 결과: `BUILD SUCCESSFUL`

## 5. 이번 단계에서 얻은 정리 포인트
- 규칙 목록은 대시보드보다 검색형 인덱스 화면으로 가는 편이 맞다.
- Mustache 셀렉트 박스는 `selected` 상태를 응답에서 미리 준비해주는 방식이 가장 덜 복잡했다.
- BOM 이슈는 규칙 모듈에서도 반복될 수 있어, 이후 파일 재작성 시 UTF-8 without BOM 저장을 계속 의식해야 한다.
