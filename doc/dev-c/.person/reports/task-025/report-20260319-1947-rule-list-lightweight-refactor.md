# 병원 규칙 목록 경량화 리팩토링 보고서

- 작업 일시: 2026-03-19 19:47 (Asia/Seoul)
- 작업 범위: 관리자 병원 규칙 목록 화면 UI 리팩토링
- 상태: 완료

## 1. 작업 배경
병원 규칙 관리 화면은 검색 바, 카드형 목록, 페이지네이션이 한 화면에 함께 들어가면서 목록 자체가 무겁게 느껴지는 상태였다. 특히 규칙 본문까지 목록에서 모두 노출되다 보니 한 번에 훑어보기 어렵고, 검색형 인덱스 화면보다 읽기 중심 화면에 가까워져 있었다.

## 2. 변경 파일
- src/main/resources/templates/admin/rule-list.mustache

## 3. 구현 내용
### 3-1. 목록 표현 방식 변경
- 카드형 목록을 직원 목록과 비슷한 테이블형 목록으로 변경했다.
- 목록 컬럼은 아래 3개만 남겼다.
  - 제목
  - 카테고리
  - 상태

### 3-2. 목록 정보 경량화
- 규칙 본문(content) 미리보기는 목록에서 제거했다.
- 제목 중심으로 빠르게 스캔할 수 있게 구성했다.
- 카테고리와 상태는 각각 배지 형태로 유지해 관리 화면 가독성을 보존했다.

### 3-3. 유지한 요소
- 상단 검색 바 구조는 그대로 유지했다.
- 조회 / 초기화 / 규칙 등록 동선은 유지했다.
- 페이지네이션 동작은 기존 구현을 그대로 사용했다.

## 4. 기대 효과
- 병원 규칙 목록이 읽기 화면이 아니라 검색형 인덱스 화면처럼 동작한다.
- 화면 전체 무게감이 줄어들어 규칙을 빠르게 찾고 상태를 확인하기 쉬워졌다.
- 직원 목록과 유사한 시각 패턴을 사용해 관리자 화면 전반의 일관성이 높아졌다.
