## 1. 작업 목표
`GET /admin/rule/list`를 단순 전체 조회 화면에서 관리용 검색형 목록으로 확장한다.
이번 단계에서는 카테고리 필터, 활성 여부 필터, 제목 키워드 검색, 페이지네이션, 최신 등록순 정렬을 백엔드 기준으로 먼저 완성하는 것이 목표였다.

## 3. 핵심 구현 내용
### 3-1. 컨트롤러 파라미터 확장
- `page`, `size`, `category`, `active`, `keyword`를 `GET /admin/rule/list`에서 수신하도록 변경했다.
- 서비스가 조합한 `AdminRuleListResponse`를 `model`로 내려주고, 기존 템플릿 호환을 위해 `rules`, `hasRules`도 함께 유지했다.

### 3-2. 서비스 정규화 로직 추가
- 잘못된 페이지/사이즈 값은 기본값(`1`, `10`)으로 보정했다.
- `category`, `active`는 잘못된 값이 들어오면 `ALL`로 안전하게 정규화했다.
- `keyword`는 trim 후 빈 문자열이면 공백 검색이 되지 않도록 정리했다.

### 3-3. Repository 검색 쿼리 추가
- 카테고리, 활성 여부, 제목 키워드를 한 번에 처리하는 `search(...)` 쿼리를 추가했다.
- 제목 검색은 `lower(title) like lower('%keyword%')` 방식으로 구현했다.
- 정렬은 `createdAt desc, id desc`로 고정했다.

### 3-4. 페이지네이션 응답 조합
- `AdminRuleListResponse`에 맞춰 현재 페이지, 전체 건수, 전체 페이지, 이전/다음 링크, 페이지 링크 목록을 생성했다.
- URL 생성 시 현재 필터/검색 조건이 유지되도록 링크를 만들었다.

### 3-5. 테스트 정리
- `AdminRuleServiceTest`
  - 필터 적용 결과
  - 잘못된 파라미터 기본값 보정
  - 페이지네이션 메타데이터
- `AdminRuleControllerTest`
  - 기본 요청
  - 필터 파라미터 전달
  - 보안 접근 제어
- 중간에 `MockMvc`에서 `request().attribute(...)` matcher로 검증하던 부분이 흔들려, `model().attribute(...)` matcher로 정리해 테스트를 안정화했다.

## 4. 작업 중 확인한 포인트
- PowerShell 출력에서는 한글이 깨져 보였지만, 실제 자바 파일은 UTF-8 기준으로 정상 저장돼 있었다.
- 반면 `task-025.md`는 실제로 인코딩이 깨져 있어서 이번 단계에서 한글을 다시 복구했다.
