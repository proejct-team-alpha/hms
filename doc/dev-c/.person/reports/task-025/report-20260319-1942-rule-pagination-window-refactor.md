## 1. 작업 배경
병원 규칙 관리 화면의 페이지네이션은 전체 페이지 번호를 모두 노출하고 있어서, 페이지 수가 늘어날수록 하단 네비게이션이 길어지고 시선이 분산되는 문제가 있었다. 이번 작업에서는 숫자 버튼을 최대 5개만 노출하는 슬라이딩 윈도우 방식으로 정리했다.

## 2. 변경 파일
- `src/main/java/com/smartclinic/hms/admin/rule/AdminRuleService.java`
- `src/test/java/com/smartclinic/hms/admin/rule/AdminRuleServiceTest.java`

## 3. 구현 내용
### 3-1. 페이지 링크 계산 로직 변경
- `AdminRuleService`의 페이지 링크 생성 로직을 전 페이지 노출 방식에서 슬라이딩 윈도우 방식으로 변경했다.
- 최대 표시 개수는 5개로 고정했다.
- 현재 페이지를 기준으로 앞/뒤 페이지가 자연스럽게 보이도록 계산했다.

### 3-2. 동작 기준
- 시작 구간: `1 2 3 4 5`
- 중간 구간: 현재 페이지를 중심으로 5개 노출
- 끝 구간: 마지막 5개 고정
- `이전`, `다음` 링크와 필터 유지 URL 구조는 기존 그대로 유지했다.

## 4. 테스트 보강
`AdminRuleServiceTest`에 아래 시나리오를 추가했다.
- 앞 구간에서 첫 5개 페이지 번호가 노출되는지 검증
- 중간 구간에서 현재 페이지 중심으로 슬라이딩되는지 검증
- 끝 구간에서 마지막 5개 페이지 번호가 노출되는지 검증

## 5. 검증 결과
- 실행 명령: `./gradlew test --tests 'com.smartclinic.hms.admin.rule.*'`
- 결과: `BUILD SUCCESSFUL`

- 페이지 수가 많아져도 하단 네비게이션이 과도하게 길어지지 않음

- 기존 검색/필터/페이지 이동 URL 구조는 그대로 유지되어 회귀 위험이 낮음
