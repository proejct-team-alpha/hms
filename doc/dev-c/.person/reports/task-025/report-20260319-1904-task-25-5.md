## 3. 핵심 보강 내용
### 3-1. 컨트롤러 렌더링 검증 추가
- 기본 목록 진입 시 아래가 실제 HTML에 포함되는지 확인했다.
  - 제목 검색
  - 카테고리
  - 활성 여부
  - 빈 결과 메시지
- 필터 적용 시 아래를 함께 검증했다.
  - 목록 row 렌더링
  - 선택된 카테고리 셀렉트
  - 선택된 활성 셀렉트
  - 키워드 입력값 유지
  - 페이지 번호 렌더링

### 3-2. 서비스 메타데이터 검증 보강
- `previousUrl`, `nextUrl`, `pageLinks`가 현재 필터 상태를 유지하는지 검증했다.
- 카테고리/활성 옵션 라벨이 화면에서 바로 쓸 수 있는 형태인지 확인했다.
- 빈 결과일 때 아래 메타데이터가 안전하게 내려오는지 검증했다.
  - `hasPages=false`
  - `pageLinks=[]`
  - `hasPrevious=false`
  - `hasNext=false`

### 3-3. 테스트 안정화 포인트
- Mustache 렌더링 결과에서 URL은 `=`가 `&#x3D;`로 이스케이프될 수 있어서, 전체 링크 문자열을 빡빡하게 비교하는 대신 페이지 번호/선택 상태/입력값 유지 같은 의미 중심 검증으로 정리했다.
- 테스트 문자열은 인코딩 흔들림을 줄이기 위해 일부는 유니코드 이스케이프 또는 ASCII 중심 값으로 정리했다.

## 4. Repository 테스트 판단
- 현재 Repository는 단일 `search(...)` 쿼리 구조로 단순하고, 서비스 테스트에서 입력 정규화와 필터 유지 URL 조합을 충분히 닫고 있다고 판단했다.
- 그래서 이번 단계에서는 별도 Repository 테스트를 추가하지 않았다.
