## 2. 핵심 코드 (Core Logic)

```java
@Test
@DisplayName("received status uses 접수 label in option and row response")
void receivedStatus_usesReceptionLabel() {
    // given
    persistReservation("RES-20260310-101", LocalDate.of(2026, 3, 10), "10:30", "RECEIVED");
    entityManager.flush();
    entityManager.clear();

    // when
    AdminReservationListResponse result = adminReservationService.getReservationList(1, 10, "RECEIVED");

    // then
    assertThat(result.selectedStatus()).isEqualTo("RECEIVED");
    assertThat(result.statusOptions())
            .filteredOn(option -> "RECEIVED".equals(option.value()))
            .singleElement()
            .satisfies(option -> {
                assertThat(option.label()).isEqualTo("접수");
                assertThat(option.selected()).isTrue();
            });
    assertThat(result.reservations()).singleElement().satisfies(item -> {
        assertThat(item.status()).isEqualTo("RECEIVED");
        assertThat(item.statusLabel()).isEqualTo("접수");
        assertThat(item.received()).isTrue();
    });
}
```

이 테스트는 단순히 상태값이 `RECEIVED`인지 보는 데서 끝나지 않고, 사용자에게 실제로 보여줄 라벨이 `접수`인지까지 함께 보장한다. 즉 내부 값과 UI 표현이 엇갈리는 회귀를 막는 역할을 한다.

## 4. 기술 딥다이브 (Technical Deep-dive)

- **왜 서비스 코드를 거의 안 건드렸는가**: 이미 [AdminReservationService.java](c:/workspace/Team/hms/src/main/java/com/smartclinic/hms/admin/reservation/AdminReservationService.java) 안에는 `RECEIVED -> 접수` 매핑이 들어 있었다. 문제는 이 규칙이 테스트로 명시돼 있지 않아서, 이후 누군가 라벨을 바꾸거나 옵션 구성을 건드려도 놓칠 수 있었다는 점이다.
- **왜 상태 옵션과 row 응답을 같이 봤는가**: 필터 드롭다운에서 `접수`라고 보여도 실제 목록 배지나 상태 텍스트가 다른 라벨을 쓸 수 있다. 반대로 row는 `접수`인데 필터 옵션은 `RECEIVED`로 남아 있으면 UX가 어색해진다. 그래서 이번 테스트는 두 위치를 한 번에 닫았다.
- **인코딩 이슈 처리**: 이번 작업에서 실제 실패 원인은 서비스 로직이 아니라 테스트 파일의 한글 기대값이 저장 과정에서 `??`로 깨진 것이었다. 그래서 기대 문자열을 바로잡는 것과 함께, [task-021.md](c:/workspace/Team/hms/doc/dev-c/task/task-021.md)도 UTF-8 기준으로 다시 저장해 작업 문서와 테스트 파일이 모두 읽히는 상태로 맞췄다.
- **왜 Task 21-2에서 끝내지 않고 문서도 같이 손봤는가**: 예약 목록 접수 통합 작업은 문서와 테스트가 같이 가야 이후 단계에서 덜 헷갈린다. 특히 이 단계는 기능을 크게 바꾸는 것이 아니라 “이미 있는 구조의 의미를 테스트와 문서로 고정”하는 작업이라, task 문서 갱신까지 같이 해두는 것이 유리하다.
