## 4. 기술 딥다이브 (Technical Deep-dive)

- **DTO 어노테이션 메시지 고정**: [CreateAdminRuleRequest.java](/c:/Users/HSystem/hms/src/main/java/com/smartclinic/hms/admin/rule/dto/CreateAdminRuleRequest.java)의 `@NotBlank`, `@Size`, `@NotNull` 메시지를 인터뷰에서 확정한 계약 문구로 직접 선언했다. 이번 범위에서는 메시지 리소스 분리보다 코드 가까이에 계약을 두는 쪽이 더 명확하다.
- **영향 범위 최소화**: 검증 규칙 자체는 바꾸지 않고 메시지 문구만 정리했기 때문에, 등록 흐름의 컨트롤러 로직이나 Mustache 바인딩 구조는 건드리지 않았다. 즉 "어떤 경우에 실패하는지"는 그대로 두고 "어떻게 안내하는지"만 바꾼 작업이다.
- **회귀 테스트 동기화**: 메시지 문자열을 직접 비교하는 [AdminRuleControllerTest.java](/c:/Users/HSystem/hms/src/test/java/com/smartclinic/hms/admin/rule/AdminRuleControllerTest.java)가 이미 있었기 때문에, 등록 실패 검증 기대값을 함께 수정해 현재 계약과 테스트가 어긋나지 않도록 맞췄다.
