private Object handleCreate(
        CreateAdminRuleRequest request,
        BindingResult bindingResult,
        RedirectAttributes redirectAttributes,
        Model model,
        HttpServletRequest req) {
    if (bindingResult.hasErrors()) {
        applyFormErrors(req, bindingResult);
        return renderForm(req, model, request);
    }
    // 등록 실패 시 service 호출 없이 같은 rule-new 화면으로 다시 렌더링한다.
}

## 4. 기술 딥다이브 (Technical Deep-dive)

- **SSR same-view 재렌더링 점검**: [AdminRuleController.java](/c:/Users/HSystem/hms/src/main/java/com/smartclinic/hms/admin/rule/AdminRuleController.java)는 등록 실패 시 이미 `renderForm(...)`을 호출한다. 이번 단계의 핵심은 "구조가 있는지"보다 "길이 초과와 category 누락 같은 실제 케이스에서도 그 구조가 끝까지 유지되는지"를 검증하는 것이었다.
- **입력값/선택값 유지 검증**: [admin/_rule-form.mustache](/c:/Users/HSystem/hms/src/main/resources/templates/admin/_rule-form.mustache)는 `titleValue`, `contentValue`, category selected 상태를 기준으로 폼을 다시 그린다. 그래서 테스트에서도 request attribute뿐 아니라 실제 렌더링 HTML에서 `value="SUPPLY" selected`와 입력값이 남아 있는지 확인하도록 했다.
- **invalid category와 required category 경계**: 기존에는 invalid category 친절 메시지 케이스가 이미 있었다. 이번 단계에서는 missing category 케이스를 별도로 추가해 `카테고리를 선택해 주세요.`가 유지되는지 확인했고, 그 결과 invalid category의 `올바른 카테고리를 선택해 주세요.`와 역할이 겹치지 않는다는 점을 테스트로 구분해 두었다.
