## 4. 기술 딥다이브 (Technical Deep-dive)

- **컨트롤러 검증 테스트 확대**: 날짜 검색은 UI에서만 보이는 기능처럼 보이지만 실제로는 컨트롤러가 기본값, 양쪽 필수, 역전 금지, 파싱 실패를 모두 책임진다. 그래서 화면 테스트도 정상 케이스만으로는 충분하지 않았다.
- **API 방어 검증**: `POST /admin/item/use`는 출고 실행 후 다시 같은 날짜 범위 로그를 반환하는 API라서, 잘못된 날짜를 받았을 때도 화면과 같은 기준으로 거절해야 한다. 이번에 400 응답 테스트를 넣어 그 규칙을 닫았다.
- **오늘 범위 보장**: `getTodayStaffUsageLogs()`는 이름만 보면 단순한 래퍼 같지만, 실제 운영에선 기본 화면 값과 직접 연결된다. 오늘 시작 시각과 다음 날 시작 시각을 쓰는 범위를 테스트로 고정해두면 시간대/로직 회귀를 막는 데 도움이 된다.
- **문자열 대신 계약 검증**: 한국어 문구는 콘솔 인코딩 환경에 따라 테스트가 불안정해질 수 있어, 이번 단계는 `dateError` 존재 여부, HTTP 상태, 서비스 호출 유무 같은 계약 중심 검증으로 안정성을 높였다.
