## 2. 핵심 전달 코드 (Core Logic)

```java
@GetMapping("/history")
public String history(@RequestParam(name = "fromDate", required = false) String fromDate,
                      @RequestParam(name = "toDate", required = false) String toDate,
                      Model model) {
    LocalDate today = LocalDate.now();
    boolean fromBlank = isBlank(fromDate);
    boolean toBlank = isBlank(toDate);
    boolean useTodayDefault = fromBlank && toBlank;
    String resolvedFromDate = useTodayDefault ? today.toString() : normalizeDateValue(fromDate);
    String resolvedToDate = useTodayDefault ? today.toString() : normalizeDateValue(toDate);

    if (fromBlank != toBlank) {
        model.addAttribute("dateError", "시작일과 종료일을 모두 입력해 주세요.");
    } else {
        LocalDate parsedFromDate = LocalDate.parse(resolvedFromDate);
        LocalDate parsedToDate = LocalDate.parse(resolvedToDate);
        histories = itemManagerService.getStockHistory(parsedFromDate, parsedToDate);
        totalIn = itemManagerService.getTotalInAmount(parsedFromDate, parsedToDate);
        totalOut = itemManagerService.getTotalOutAmount(parsedFromDate, parsedToDate);
    }
}

public List<ItemStockLogDto> getStockHistory(LocalDate fromDate, LocalDate toDate) {
    return stockLogRepository.findByCreatedAtGreaterThanEqualAndCreatedAtLessThanOrderByCreatedAtDesc(
                    fromDate.atStartOfDay(),
                    toDate.plusDays(1).atStartOfDay())
            .stream()
            .map(ItemStockLogDto::new)
            .toList();
}
```

## 3. 쉬운 비유로 설명하면 (Easy Analogy)

- 예전에는 장부를 처음부터 끝까지 다 펼쳐서 보여줬다면, 지금은 “오늘 장부만 먼저 펼쳐주고”, 사용자가 기간을 바꾸면 그 범위만 다시 꺼내 보여주는 구조가 됐습니다.
- 그래서 화면 검색이 조금 더 실무형으로 가까워졌고, 뒤에서 조회하는 서비스도 같은 범위를 기준으로 합계를 계산하게 됐습니다.

## 4. 기술 딥다이브 (Technical Deep-dive)

- **기본값과 검증 분리**: 날짜가 둘 다 없을 때만 `오늘 ~ 오늘`로 기본값을 넣고, 한쪽만 빠진 경우엔 자동 보정하지 않고 검증 에러로 분기했습니다. 이렇게 해야 사용자가 의도치 않게 조회 범위를 오해하지 않습니다.
- **반개구간 날짜 범위 처리**: `toDate` 당일 23:59:59를 계산하는 대신 `toDate.plusDays(1).atStartOfDay()` 직전까지를 범위로 보는 방식으로 구현했습니다. datetime 로그 조건에서 가장 덜 흔들리는 패턴입니다.
- **합계도 같은 범위를 기준으로 계산**: 단순 목록만 줄이는 게 아니라 `totalIn`, `totalOut`도 같은 날짜 범위 기준으로 다시 계산하도록 맞췄습니다. 화면 숫자와 목록 데이터가 서로 다른 범위를 보지 않게 하려는 목적입니다.
- **테스트 위치 보정**: 기존 물품 서비스 테스트는 `item` 패키지 아래에 있었고, admin SSR 쪽은 테스트가 없어서 이번에 `AdminItemControllerTest`를 새로 추가했습니다. 덕분에 컨트롤러 기본값/검증 흐름과 서비스 날짜 경계를 각각 분리해서 확인할 수 있게 됐습니다.
