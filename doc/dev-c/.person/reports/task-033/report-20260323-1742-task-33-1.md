## 2. 핵심 전달 코드 (Core Logic)

```java
// AdminItemController.history(): 현재는 날짜 조건 없이 통합 입출고 이력만 내려줍니다.
@GetMapping("/history")
public String history(Model model) {
    List<?> histories = itemManagerService.getStockHistory();
    model.addAttribute("histories", histories);
    model.addAttribute("hasHistories", !histories.isEmpty());
    model.addAttribute("totalCount", histories.size());
    model.addAttribute("totalIn", itemManagerService.getTotalInAmount());
    model.addAttribute("totalOut", itemManagerService.getTotalOutAmount());
    model.addAttribute("pageTitle", "입출고 내역");
    model.addAttribute("isAdminItemHistory", true);
    return "admin/item-history";
}

// ItemManagerService.getStockHistory(): 전체 이력을 최신순으로 가져오는 구조입니다.
public List<ItemStockLogDto> getStockHistory() {
    return stockLogRepository.findAllByOrderByCreatedAtDesc()
            .stream().map(ItemStockLogDto::new).toList();
}

// ItemUsageLogRepository: 출고 로그는 이미 usedAt 기간 조회 메서드를 갖고 있습니다.
List<ItemUsageLog> findByReservationIdIsNullAndUsedAtBetweenOrderByUsedAtDesc(
        LocalDateTime start, LocalDateTime end);
```

- 지금 구조는 `item-history`가 통합 장부이고, `item-use`는 장부 조회 화면이라기보다 오늘 출고를 처리하는 작업대에 가깝습니다.
- 그래서 날짜 검색을 붙일 때도 단순히 두 화면에 같은 필터를 복붙하면 안 되고, 어떤 화면이 조회 책임을 갖는지 먼저 나눠서 보는 게 맞습니다.

## 4. 기술 딥다이브 (Technical Deep-dive)

- **통합 입출고 이력 구조**: 관리자 입출고 이력은 `ItemStockLog`를 기준으로 IN/OUT을 함께 보여주는 구조입니다. 그래서 날짜 검색 1차 대상은 `GET /admin/item/history`가 됩니다.
- **출고 화면의 실제 책임**: `GET /admin/item/use`는 오늘 출고 실행을 위한 화면이고, `todayLogs`도 오늘 데이터만 보여줍니다. 이 화면에 날짜 검색을 붙이려면 “실행 화면 안의 로그 확장”으로 볼지, “출고 이력 조회 화면으로 확장”할지 먼저 정해야 합니다.
- **Repository 재사용 가능성**: `ItemStockLogRepository`는 이미 `createdAtBetween` 메서드를 가지고 있고, `ItemUsageLogRepository`도 `usedAtBetween` 메서드를 갖고 있어서 날짜 검색 자체는 완전히 새로 설계하지 않아도 됩니다.
- **DTO 차이로 인한 후속 영향**: `ItemStockLogDto`는 날짜와 시간을 모두 보여주지만, `ItemUsageLogDto`는 시간만 노출합니다. 출고 내역을 기간 검색 화면처럼 다루려면 DTO가 날짜 표현을 더 가져야 할 수 있습니다.
- **테스트 위치 주의점**: admin item 기능이지만 테스트는 `admin.item`이 아니라 `src/test/java/com/smartclinic/hms/item` 아래에 있습니다. 다음 단계에서 테스트를 보강할 때 이 구조를 그대로 따라가거나, admin 기준 테스트를 새로 만들지 판단이 필요합니다.
