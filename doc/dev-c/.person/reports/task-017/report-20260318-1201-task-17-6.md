6. 이 문제는 테스트만 바꾸는 것으로 덮지 않고, 응답 모델 [AdminDepartmentListResponse.java](/c:/workspace/Team/hms/src/main/java/com/smartclinic/hms/admin/department/AdminDepartmentListResponse.java)에 `hasPages` 플래그를 추가해 서버가 페이지 존재 여부를 명시적으로 알려주도록 정리했다.
7. 그리고 [AdminDepartmentService.java](/c:/workspace/Team/hms/src/main/java/com/smartclinic/hms/admin/department/AdminDepartmentService.java)에서 `totalPages > 0`일 때만 `hasPages=true`가 되게 만들고, [department-list.mustache](/c:/workspace/Team/hms/src/main/resources/templates/admin/department-list.mustache)는 이제 `{{#model.hasPages}}`를 기준으로 페이지 정보를 렌더링하도록 변경했다.
8. 마지막으로 `./gradlew test --tests 'com.smartclinic.hms.admin.department.*'`를 실행해 `admin.department` 범위 테스트가 모두 통과하는 것을 확인하고 작업을 마무리했다.

## 2. 핵심 전달 코드 (Core Logic)

```java
return new AdminDepartmentListResponse(
        departments,
        buildPageLinks(totalPages, currentPage, safeSize),
        pageResult.getTotalElements(),
        currentPage,
        safeSize,
        totalPages,
        totalPages > 0,
        hasPrevious,
        hasNext,
        hasPrevious ? buildListUrl(currentPage - 1, safeSize) : "",
        hasNext ? buildListUrl(currentPage + 1, safeSize) : ""
);
```

```mustache
{{#model.hasPages}}
  <div>{{model.currentPage}} / {{model.totalPages}}페이지</div>
{{/model.hasPages}}
{{^model.hasPages}}
  <div>0 / 0페이지</div>
{{/model.hasPages}}
```

이번 작업의 핵심은 “페이지가 있는지 없는지”를 템플릿이 숫자 `0`에 기대서 추측하게 두지 않고, 서버가 `hasPages`라는 명시적인 의미값으로 전달하게 바꾼 것이다. 덕분에 빈 목록과 일반 목록이 훨씬 안정적으로 구분된다.

## 4. 기술 딥다이브 (Technical Deep-dive)

- **테스트가 회귀를 발견하는 방식**: 이번 작업은 단순히 테스트 개수를 늘린 것이 아니라, 사용자 화면에서 실제로 보이는 결과를 기준으로 테스트를 추가했다. 특히 `빈 목록`처럼 평소 자주 보지 않는 경계 케이스를 검증하자, 기존 구현에서 숨어 있던 렌더링 문제를 바로 찾을 수 있었다.
- **Mustache와 숫자 조건의 애매함**: 템플릿 엔진에서 숫자 `0`이 항상 우리가 기대하는 방식으로 truthy/falsey처럼 다뤄지지 않을 수 있다. 그래서 `totalPages` 숫자 자체를 분기 기준으로 삼기보다, 서버가 `hasPages`라는 boolean을 만들어 주는 게 더 안전하다. 이건 템플릿이 숫자의 의미를 해석하지 않고, 이미 해석된 결과만 소비하게 만드는 방식이다.
- **테스트 주도 버그 수정**: 이번엔 테스트를 추가하는 과정에서 화면 버그가 발견됐고, 그 버그를 서비스 응답 모델과 템플릿 계약까지 포함해서 함께 정리했다. 즉 테스트 보강이 단순한 “검증 추가”를 넘어서, 화면 계약을 더 단단하게 만드는 역할까지 한 셈이다.
