## 2. 핵심 전달 코드 (Core Logic)

```java
public AdminDepartmentListResponse getDepartmentList(int page, int size) {
    int safePage = page < 1 ? DEFAULT_PAGE : page;
    int safeSize = size < 1 ? DEFAULT_SIZE : size;

    Pageable pageable = PageRequest.of(safePage - 1, safeSize, Sort.by(Sort.Order.asc("name")));
    Page<Department> pageResult = adminDepartmentRepository.findAllByOrderByNameAsc(pageable);

    List<AdminDepartmentDto> departments = pageResult.getContent().stream()
            .map(AdminDepartmentDto::new)
            .collect(Collectors.toList());

    int currentPage = pageResult.getNumber() + 1;
    int totalPages = pageResult.getTotalPages();
    boolean hasPrevious = pageResult.hasPrevious();
    boolean hasNext = pageResult.hasNext();

    return new AdminDepartmentListResponse(
            departments,
            buildPageLinks(totalPages, currentPage, safeSize),
            pageResult.getTotalElements(),
            currentPage,
            safeSize,
            totalPages,
            hasPrevious,
            hasNext,
            hasPrevious ? buildListUrl(currentPage - 1, safeSize) : "",
            hasNext ? buildListUrl(currentPage + 1, safeSize) : ""
    );
}
```

## 4. 기술 딥다이브 (Technical Deep-dive)

- **Spring Data JPA 페이징**: 이번 작업의 중심은 `Pageable`과 `Page<T>`를 도입한 것이다. `PageRequest.of(safePage - 1, safeSize, Sort.by(...))`로 페이징 요청을 만들고, JPA가 반환한 `Page<Department>`에서 총 건수와 현재 페이지, 이전/다음 페이지 유무를 읽어왔다.
- **SSR 화면용 응답 모델 분리**: 단순히 `List<Department>`만 넘기면 템플릿이 페이지네이션 정보를 알 수 없다. 그래서 `AdminDepartmentListResponse`를 추가해 목록 데이터와 페이지 메타정보를 한 묶음으로 전달하는 구조로 바꿨다.
- **기본값 보정 로직**: 잘못된 `page`, `size`가 들어와도 화면이 깨지지 않도록 `page < 1`, `size < 1`이면 각각 `1`, `10`으로 보정했다. 이 로직을 서비스에 두면 컨트롤러가 얇게 유지되고, API나 다른 호출 경로가 생겨도 재사용하기 쉽다.
- **화면 계약 유지와 최소 템플릿 수정**: 원래 Mustache는 `departments`를 바로 읽고 있었지만, 이번부터 컨트롤러가 `model` 하나로 묶어 넘기기 때문에 `model.departments`로 바꿨다. 이건 사실 `Task 17-4`의 일부와 경계가 닿아 있지만, 이 수정이 없으면 백엔드 페이징 구조를 넣는 순간 화면이 바로 깨지기 때문에 최소 범위로 같이 반영했다.
- **컨트롤러/서비스 테스트 분리**: 서비스 테스트는 계산 로직(기본값 보정, 페이지 링크 생성)을 검증하고, 컨트롤러 테스트는 `page`, `size` 파라미터 전달과 `model + pageTitle` 렌더링 계약을 검증한다. 이렇게 나누면 다음 단계 UI 작업에서도 백엔드 쪽 회귀를 빠르게 잡아낼 수 있다.
