## 4. 기술 딥다이브 (Technical Deep-dive)

- **Mustache SSR 렌더링**: 이번 화면은 클라이언트에서 데이터를 다시 조립하는 방식이 아니라 서버가 템플릿에 값을 넣어 HTML을 완성하는 구조다. 그래서 템플릿 문구만 고치는 것으로 끝나지 않고, 컨트롤러의 `pageTitle`과 DTO의 `activeText`도 같이 맞춰야 화면이 정상적으로 복구된다.
- **UTF-8 인코딩 복구**: 한글이 깨진 원인은 템플릿 문자열과 일부 Java 문자열이 정상 인코딩으로 보존되지 않은 상태였기 때문이다. 파일을 UTF-8 기준으로 다시 저장하면서 사용자 노출 텍스트를 한국어로 복구했다.
- **서버-템플릿 데이터 계약 유지**: 이번 단계에서는 페이징이나 조회 책임을 건드리지 않고, 기존 데이터 계약인 `departments`, `hasDepartments`, `pageTitle`, `activeText`, `activeBadgeClass`를 유지한 채 화면만 복구했다. 덕분에 `Task 17-2` 이후 작업과도 충돌하지 않는다.
- **안전한 검증 방식**: 진료과 전용 테스트가 아직 없는 상태라, 회귀 여부를 확인하기 위해 전체 테스트를 실행했다. 이 방식은 작업 범위가 작더라도 다른 모듈에 영향이 없는지 한 번에 확인하는 데 유효했다.
