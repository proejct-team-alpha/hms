# 이번 작업 보고서: 직원 등록/수정 SSR 화면에 퇴사 일시와 잠금 규칙 반영

## 1. 전체 작업 흐름 (Workflow)

1. 먼저 `AdminStaffController`, `AdminStaffService`, `staff-form.mustache`, `AdminStaffControllerTest`, `AdminStaffServiceTest`를 다시 확인해 현재 SSR 폼이 어디까지 바뀌어 있는지 점검했다.
2. 점검 결과 `퇴사 일시` 필드는 DTO/폼 응답 쪽에만 일부 추가돼 있었고, 실제 템플릿은 아직 예전 구조를 유지하고 있었다. 또한 테스트는 새 시그니처를 반영하지 못해 그대로는 컴파일이 깨질 상태였다.
3. 그래서 이번 단계에서는 화면 책임만 먼저 닫는다는 기준으로 접근했다. `퇴사 일시` 입력을 SSR 폼에 추가하고, 본인 계정과 비활성 직원에 대한 잠금 상태를 `AdminStaffFormResponse` 기반으로 렌더링하도록 정리했다.
4. 본인 계정처럼 disabled 필드가 필요한 경우 값이 서버에 안 넘어가는 문제를 막기 위해 hidden input을 같이 두는 방식으로 정리했다.
5. 마지막으로 `AdminStaffControllerTest`, `AdminStaffServiceTest`, `AdminStaffApiController`까지 컴파일 의존 지점을 같이 정리하고 `admin.staff` 범위 테스트를 다시 실행해 회귀가 없는지 확인했다.

## 2. 핵심 전달 코드 (Core Logic)

```html
<select
  id="staff-active"
  name="active"
  {{#model.employmentStatusLocked}}disabled{{/model.employmentStatusLocked}}
>
  {{#model.employmentStatusOptions}}
    <option value="{{value}}" {{#selected}}selected{{/selected}}>{{label}}</option>
  {{/model.employmentStatusOptions}}
</select>
{{#model.employmentStatusLocked}}
  <input type="hidden" name="active" value="{{#model.active}}true{{/model.active}}{{^model.active}}false{{/model.active}}">
{{/model.employmentStatusLocked}}

<input
  type="datetime-local"
  id="staff-retired-at"
  name="retiredAt"
  value="{{model.retiredAt}}"
  {{#model.retiredAtLocked}}disabled{{/model.retiredAtLocked}}
/>
{{#model.retiredAtLocked}}
  <input type="hidden" name="retiredAt" value="{{model.retiredAt}}">
{{/model.retiredAtLocked}}
```

- `employmentStatusLocked`, `retiredAtLocked`를 그대로 템플릿 조건으로 써서 SSR에서 잠금 상태를 렌더링했다.
- disabled 필드는 submit 시 값이 빠지기 때문에, hidden input을 함께 두어 서버가 현재 값을 계속 받을 수 있게 했다.
- 이 방식 덕분에 Task 35-5에서 실제 저장 규칙을 서비스에 넣을 때도 화면 구조를 다시 흔들 필요가 없어진다.

## 3. 쉬운 비유 (Easy Analogy)

- 이번 작업은 "수정 가능한 칸과 읽기 전용 칸을 구분한 인사 카드"를 만드는 것과 비슷하다.
- 예를 들어 본인 카드에서는 `재직 상태`와 `퇴사 일시` 칸에 자물쇠를 걸어두고, 다른 관리자만 그 칸을 만질 수 있게 둔 셈이다.
- 다만 카드에 적힌 값 자체는 제출할 때 사라지면 안 되므로, 보이지 않는 복사본(hidden input)을 같이 넣어 둔 구조다.

## 4. 기술 딥다이브 (Technical Deep-dive)

- **SSR same-view 잠금 표현**: 이번 단계에서는 비즈니스 규칙을 서비스에 넣기 전에, 화면이 먼저 정책을 설명할 수 있어야 했다. 그래서 `AdminStaffFormResponse`에 `selfEdit`, `employmentStatusLocked`, `retiredAtLocked`, `retiredAt` 값을 추가하고 템플릿이 그 값을 직접 읽도록 구성했다.
- **disabled 필드와 hidden input**: HTML 폼에서는 disabled 필드가 submit payload에 포함되지 않는다. 본인 계정이나 비활성 직원처럼 `재직 상태`를 잠가야 하는 경우, hidden input을 같이 두지 않으면 서버에서 값이 비어 버린다. 이번 구현은 이 문제를 먼저 막아 둔 것이다.
- **컴파일 의존 최소 정리**: `UpdateAdminStaffRequest`에 `retiredAt`이 추가되면서 `AdminStaffApiController`와 `AdminStaffServiceTest`도 같이 깨질 수 있었다. 아직 API 규칙 반영 단계는 아니지만, `null` 위임과 테스트 시그니처 보정으로 브랜치 전체 컴파일이 흔들리지 않게 맞췄다.
