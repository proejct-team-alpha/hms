# 작업 보고서 [Task 36-3 직원 수정 화면 UI 리팩토링]

- **작업 일시**: 2026-03-24
- **진행 상태**: 완료

## 1. 전체 작업 흐름 (Workflow)

1. 직원 수정 화면에서 `재직 상태`와 `퇴사 일시`가 서로 떨어져 있어 한 번에 읽히지 않는 문제를 먼저 확인했습니다.
2. `퇴사 일시`를 수정 화면에서만 먼저 `날짜 + 시간` 입력으로 분리하고, 등록 화면은 기존 `datetime-local`을 유지하는 범위로 작업을 고정했습니다.
3. `AdminStaffFormResponse`에 `retiredAtDate`, `retiredAtHour`, `retiredAtHourOptions`를 추가해 수정 화면 렌더링에 필요한 값을 명시적으로 내려주도록 정리했습니다.
4. `AdminStaffService`에서 수정 화면용 `퇴사 일시` 날짜/시간 포맷과 24시간 옵션 리스트를 만들도록 보강했습니다.
5. `staff-form.mustache`에서 수정 화면만 `재직 상태` 옆에 `퇴사 일시`를 두고, 날짜 input + 시간 select + hidden input 구조로 바꿨습니다.
6. 제출 직전 브라우저 스크립트에서 `retiredAtDate`와 `retiredAtHour`를 `yyyy-MM-ddTHH:00` 형식의 hidden `retiredAt`으로 합치도록 연결했습니다.
7. 손상된 `AdminStaffControllerTest`를 현재 템플릿 구조 기준으로 다시 작성하고, `admin.staff` 범위 테스트가 다시 통과하는지 확인했습니다.

## 2. 핵심 전달 코드 (Core Logic)

```html
<div class="grid grid-cols-[minmax(0,1fr)_120px] gap-3">
  <input type="date" id="staff-retired-at-date" name="retiredAtDate" value="{{model.retiredAtDate}}" />
  <select id="staff-retired-at-hour" name="retiredAtHour">
    <option value="">시</option>
    {{#model.retiredAtHourOptions}}
      <option value="{{value}}" {{#selected}}selected{{/selected}}>{{label}}</option>
    {{/model.retiredAtHourOptions}}
  </select>
</div>
<input type="hidden" id="staff-retired-at-hidden" name="retiredAt" value="{{model.retiredAt}}">
```

```js
const syncRetiredAtHidden = () => {
  const dateValue = retiredAtDateInput.value;
  const hourValue = retiredAtHourSelect.value;

  if (!dateValue && !hourValue) {
    retiredAtHiddenInput.value = '';
    return true;
  }

  if (!dateValue || !hourValue) {
    retiredAtHiddenInput.value = '';
    retiredAtDateInput.setCustomValidity('퇴사 일시는 날짜와 시간을 함께 선택해 주세요.');
    retiredAtHourSelect.setCustomValidity('퇴사 일시는 날짜와 시간을 함께 선택해 주세요.');
    return false;
  }

  retiredAtHiddenInput.value = `${dateValue}T${hourValue}:00`;
  return true;
};
```

이 코드의 핵심은 화면에서는 분을 직접 받지 않되, 서버가 기존 `LocalDateTime retiredAt` 계약을 그대로 받을 수 있게 hidden 값으로 조합해 준다는 점입니다.

## 3. 쉬운 비유 (Easy Analogy)

- 이번 작업은 `퇴사 일시`를 “달력 + 시간표”로 나눠서 받는 것과 비슷합니다.
- 예전에는 사용자가 시계의 분까지 직접 맞춰야 했는데, 실제로는 우리 프로젝트에서 분 단위가 중요하지 않았습니다.
- 그래서 날짜를 고르고, 몇 시인지 고른 다음, 시스템이 뒤에서 자동으로 `15:00`처럼 정확한 저장값을 만들어 주도록 바꾼 셈입니다.

## 4. 기술 딥다이브 (Technical Deep-dive)

- **SSR 화면과 저장 계약 분리**: 화면은 `retiredAtDate`와 `retiredAtHour`로 단순화하고, 저장은 기존 `retiredAt` hidden 필드로 이어주어 서버 계약을 한 번에 깨지 않도록 했습니다.
- **폼 검증의 브라우저 선처리**: 날짜나 시간 하나만 고른 채 submit 하는 실수를 브라우저에서 먼저 막아, 서버까지 잘못된 값이 들어가기 전에 UX를 정리했습니다.
- **수정 화면 한정 리팩토링**: 등록 화면까지 한 번에 바꾸지 않고, 이번 단계는 수정 화면에만 적용해 영향 범위를 좁혔습니다. 덕분에 Task 36-4에서 등록 화면을 별도 단계로 이어갈 수 있습니다.
- **테스트 복구와 현재 구조 고정**: 깨진 컨트롤러 테스트를 현재 템플릿 구조 기준으로 다시 작성하면서, `staff-retired-at-date`, `staff-retired-at-hour`, hidden `retiredAt` 같은 새 구조를 테스트로 고정했습니다.
