# 작업 보고서 [Task 36-4 직원 등록 화면 입력 형식 반영]

- **작업 일시**: 2026-03-24
- **진행 상태**: 완료

## 1. 전체 작업 흐름 (Workflow)

1. 등록 화면은 레이아웃은 유지하고 `퇴사 일시` 입력 형식만 바꾸자는 기준을 다시 확인했습니다.
2. 수정 화면에서 먼저 적용한 `날짜 input + 시간 select + hidden retiredAt` 패턴을 등록 화면에도 그대로 옮기는 쪽이 가장 안전하다고 판단했습니다.
3. `staff-form.mustache`의 등록 화면 블록에서 기존 `datetime-local` 입력을 제거하고, 날짜/시간 분리 입력으로 교체했습니다.
4. hidden `retiredAt` 필드는 유지해서 서버 계약은 그대로 두고, 브라우저 스크립트가 `yyyy-MM-ddTHH:00` 형식으로 값을 채우게 했습니다.
5. `AdminStaffControllerTest`에서 등록 화면도 새 구조를 렌더링하는지 확인하도록 기대값을 업데이트했습니다.
6. 마지막으로 `admin.staff` 범위 테스트를 다시 실행해 등록/수정 화면이 함께 회귀 없이 동작하는지 검증했습니다.

## 2. 핵심 전달 코드 (Core Logic)

```html
<input type="hidden" id="staff-retired-at-hidden" name="retiredAt" value="{{model.retiredAt}}">
<div class="grid grid-cols-[minmax(0,1fr)_120px] gap-3">
  <input type="date" id="staff-retired-at-date" name="retiredAtDate" value="{{model.retiredAtDate}}" />
  <select id="staff-retired-at-hour" name="retiredAtHour">
    <option value="">시</option>
    {{#model.retiredAtHourOptions}}
      <option value="{{value}}" {{#selected}}selected{{/selected}}>{{label}}</option>
    {{/model.retiredAtHourOptions}}
  </select>
</div>
```

```js
if (!dateValue && !hourValue) {
  retiredAtHiddenInput.value = '';
  return true;
}

if (!dateValue || !hourValue) {
  retiredAtHiddenInput.value = '';
  retiredAtDateInput.setCustomValidity('퇴사 일시는 날짜와 시간을 함께 선택해 주세요.');
  retiredAtHourSelect.setCustomValidity('퇴사 일시는 날짜와 시간을 함께 선택해 주세요.');
  return false;
}

retiredAtHiddenInput.value = `${dateValue}T${hourValue}:00`;
```

등록 화면도 이제 수정 화면과 같은 규칙으로 값을 만들기 때문에, 저장 시 분이 항상 `00`으로 고정됩니다.

## 3. 쉬운 비유 (Easy Analogy)

- 이번 변경은 등록 화면의 `퇴사 일시` 입력을 “복잡한 디지털 시계”에서 “달력 + 정각 버튼”으로 바꾼 것과 비슷합니다.
- 사용자는 날짜를 고르고 몇 시인지 고르기만 하면 되고, 시스템이 뒤에서 자동으로 `18:00` 같은 저장용 값을 만들어 줍니다.
- 그래서 입력은 쉬워지고, 저장 규칙은 더 일관돼집니다.

## 4. 기술 딥다이브 (Technical Deep-dive)

- **서버 계약 유지, 화면 입력만 변경**: 이번 단계는 DTO를 다시 설계하지 않고, hidden `retiredAt`을 유지하는 방식으로 화면 입력 형식만 바꿨습니다. 그래서 영향 범위를 줄이면서 UX 개선을 먼저 가져갈 수 있었습니다.
- **등록/수정 공용 스크립트 재사용**: `staff-form.mustache` 하단의 `syncRetiredAtHidden()` 로직이 등록/수정 양쪽 모두에 동일하게 동작하도록 만들어 중복 JS를 피했습니다.
- **템플릿 공용화 유지**: 등록과 수정은 같은 `staff-form.mustache`를 계속 쓰되, `{{^model.editMode}}` 블록만 교체해서 레이아웃 톤을 유지했습니다.
- **테스트로 렌더링 구조 고정**: `AdminStaffControllerTest`에서 등록 화면도 `staff-retired-at-date`, `staff-retired-at-hour`, hidden `retiredAt`를 렌더링하는지 검증해 새 입력 구조를 회귀 방지했습니다.
