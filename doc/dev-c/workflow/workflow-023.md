# 환자 정보 부분 수정 REST API 명세서 (workflow-023)

## 문제 정의
관리자 영역에서 환자 정보를 JSON 기반으로 부분 수정할 수 있는 REST API를 구현한다. 기존 SSR 환자 목록/상세 화면은 유지하되, 향후 비동기 수정 UI 또는 관리자 도구에서 재사용할 수 있도록 `POST /admin/api/patients/{id}/update` API를 제공한다.

## 현재 판단
- 환자 목록/상세 SSR 화면은 이미 구현되어 있고, 현재는 조회 중심이다.
- 환자 수정 기능은 아직 별도 SSR 폼보다 API로 먼저 여는 편이 확장에 유리하다.
- 현재 프로젝트에서 환자 동일성 판단은 사실상 전화번호 기준으로 동작하므로, 환자 정보 수정에서도 전화번호 중복 방지가 중요하다.
- 응답 형식은 기존 관리자 API 패턴과 동일하게 `Resp.ok(...)` 공통 래퍼를 유지하는 것이 적절하다.

## 대상 사용자
- 관리자(`ROLE_ADMIN`)

## 확정 API
- `POST /admin/api/patients/{id}/update`

## 수정 가능 필드
- `name`
- `phone`
- `note`

## 응답 형식
- 기존 프로젝트 공통 응답 규격 유지
- `Resp.ok(...)` 사용
- 성공 시 수정된 환자 정보 DTO를 body에 담아 반환

## 전달 요구사항
1. `POST /admin/api/patients/{id}/update` 엔드포인트를 추가한다.
2. 요청은 JSON 기반으로 받는다.
3. 수정 가능한 필드는 아래로 제한한다.
   - 이름(`name`)
   - 연락처(`phone`)
   - 메모(`note`)
4. 성공 시 수정된 환자 정보를 응답 DTO로 반환한다.
5. 응답은 `Resp.ok(...)` 공통 래퍼를 사용한다.
6. 관리자 권한 기준으로만 접근 가능해야 한다.

## 검증 규칙
1. `name`
   - `@NotBlank`
2. `phone`
   - `@NotBlank`
3. `note`
   - 선택 입력
4. 전화번호 중복 방지
   - 다른 환자와 동일한 전화번호로 수정하려는 경우 차단한다.

## 이번 단계에서 하지 않는 것
- 환자 수정 SSR 폼 추가
- 환자 삭제 API 추가
- 이메일/주소 수정 API 확장
- 전화번호 패턴/길이 검증 고도화
- 환자 수정 이력 관리 기능 추가

## 제약 조건
- 기존 URL prefix(`/admin/api/**`)를 유지한다.
- 기존 `Resp`, `CustomException`, `GlobalExceptionHandler` 포맷을 유지한다.
- 입력 DTO는 `@Valid` 기반으로 검증한다.
- 수정 대상이 없는 경우 프로젝트 규칙에 맞는 not found 예외를 사용한다.
- 전화번호 중복 검사는 자기 자신을 제외한 다른 환자 기준으로 수행한다.
- 현재 환자 조회/상세 SSR 기능과 충돌하지 않도록 `admin.patient` 범위에 책임을 둔다.

## 엣지 케이스
- 존재하지 않는 `patientId` 요청
- `name` 공백 입력
- `phone` 공백 입력
- 다른 환자와 전화번호 중복
- `note`만 수정하고 `name`, `phone`은 동일 값으로 유지하는 요청
- 기존 전화번호와 동일한 값으로 다시 저장하는 경우는 정상 허용해야 한다.

## 수용 기준
- [ ] `POST /admin/api/patients/{id}/update`가 동작한다.
- [ ] `name`, `phone`, `note`를 수정할 수 있다.
- [ ] `name`, `phone` 공백 입력은 차단된다.
- [ ] 다른 환자와 전화번호 중복 시 수정이 차단된다.
- [ ] 성공 시 `Resp.ok(...)` 형식으로 수정 결과를 반환한다.
- [ ] 관련 API 테스트가 통과한다.

## 구현 순서
1. 현재 `admin.patient` 수정 가능 구조와 중복 판단 기준 점검
2. 수정 요청 DTO와 응답 DTO 설계
3. 서비스 수정 로직 구현
4. API 컨트롤러 추가 또는 기존 컨트롤러 확장
5. 전화번호 중복/없는 환자/기본 검증 테스트 보강
6. 범위 테스트 및 전체 테스트 확인

- 환자 수정 SSR 폼 추가
