# Workflow 033 - 입출고 내역 날짜 기간 검색

## 문제 정의
관리자 물품 입출고 내역 화면에서 날짜를 지정해 검색할 수 있도록 기능을 추가한다. 입출고 업무는 날짜가 핵심 축이므로, 기존 검색 조건과 함께 사용할 수 있는 기간 검색 구조가 필요하다.

## 대상 화면
- `GET /admin/item/history`
- `GET /admin/item/use`

## 현재 판단
- `GET /admin/item/history`는 `ItemStockLog` 기준의 통합 입출고 이력 화면이다.
- `GET /admin/item/use`는 출고 실행 화면이지만, 상단 `todayLogs` 영역을 통해 날짜 기준 출고 내역 조회를 함께 제공할 수 있다.
- 따라서 날짜 검색은 두 화면 모두 지원하되,
  - `item-history`는 서버 날짜 필터 + 기존 클라이언트 검색/구분 필터
  - `item-use`는 서버 날짜 필터 + 기존 클라이언트 물품 검색/카테고리 필터
  조합으로 정리하는 것이 가장 자연스럽다.

## 확정 방향
1. 입고 내역과 출고 내역 모두 날짜 검색을 지원한다.
2. 날짜 검색은 기간 검색(`fromDate ~ toDate`)으로 구현한다.
3. 기존 검색/필터 조건과 날짜 검색은 함께 유지된다.
4. 기본값은 `오늘 ~ 오늘`이다.
5. 날짜는 양쪽 모두 입력해야 하며, 한쪽만 입력하면 검증 에러를 준다.
6. `fromDate > toDate`는 검증 에러로 처리한다.
7. 검색 UI는 기존 관리자 검색 바 패턴에 맞춰 상단에 배치한다.

## 날짜 검색 규칙
### 기본값
- `fromDate = 오늘`
- `toDate = 오늘`

### 검증 규칙
- `fromDate`만 있고 `toDate`가 없으면 에러
- `toDate`만 있고 `fromDate`가 없으면 에러
- `fromDate > toDate`면 에러
- 날짜 형식이 잘못되면 에러
- 정상 범위일 때만 조회 수행

### 조회 기준
- 단일일 조회: `fromDate == toDate`
- 기간 조회: `fromDate ~ toDate`
- datetime 범위는 `[fromDate.atStartOfDay(), toDate.plusDays(1).atStartOfDay())`

## UI 방향
- 두 화면 모두 상단에 같은 형태의 날짜 검색 바를 둔다.
- 구성 요소
  - 시작일
  - 종료일
  - 조회 버튼
  - 초기화 버튼
- 초기화는 각 화면의 기본 URL로 이동시키고, 컨트롤러가 `오늘 ~ 오늘` 기본값을 다시 주입한다.
- 기존 클라이언트 검색/필터 상태(`keyword`, `type`, `category`)는 hidden field + URL query string으로 유지한다.

## 구현 범위
### 1. 컨트롤러
- `fromDate`, `toDate`를 받아 날짜 범위를 해석하고 검증한다.
- 기본 진입 시 `오늘 ~ 오늘`을 모델에 넣는다.
- 검증 실패 시 same-view에서 `dateError`를 보여준다.
- `POST /admin/item/use`도 현재 날짜 범위를 받아 같은 기준으로 로그를 반환한다.

### 2. 서비스
- 날짜 범위 조건을 포함한 입출고 로그 조회 메서드를 제공한다.
- 총 입고량, 총 출고량, 총 출고 건수 계산을 날짜 범위 기준으로 제공한다.

### 3. 리포지토리
- `ItemStockLogRepository`, `ItemUsageLogRepository`에 날짜 범위 조회/합계 메서드를 둔다.
- 서버 페이지네이션이 아니므로 정렬과 범위 필터에 집중한다.

### 4. 템플릿
- `item-history.mustache`, `item-use.mustache`에 날짜 검색 바를 추가한다.
- `fromDate`, `toDate`를 SSR 값으로 렌더링한다.
- 클라이언트 필터 상태를 hidden field와 URL 상태로 동기화한다.

### 5. 테스트
- 기본값 `오늘 ~ 오늘`
- 한쪽 날짜 누락
- 날짜 역전
- 날짜 형식 오류
- 날짜 범위 조회
- AJAX 출고 API의 날짜 범위 유지
- 클라이언트 상태 유지용 hidden field 렌더링

## 수용 기준
- [x] 입고 내역 조회에서 날짜 기간 검색이 가능하다.
- [x] 출고 내역 조회에서 날짜 기간 검색이 가능하다.
- [x] 초기 진입 시 `오늘 ~ 오늘`이 기본값으로 세팅된다.
- [x] 날짜 한쪽만 입력하면 검증 에러가 표시된다.
- [x] `fromDate > toDate`면 검증 에러가 표시된다.
- [x] 날짜 형식 오류 시 검증 에러가 표시된다.
- [x] 기존 검색 조건과 날짜 조건을 함께 유지할 수 있다.
- [x] 관련 컨트롤러/서비스/리포지토리/테스트가 통과한다.

## 구현 점검 결과
- `item-history`는 날짜 범위에 맞는 통합 입출고 이력, 총 입고량, 총 출고량을 서버에서 계산한다.
- `item-use`는 날짜 범위에 맞는 출고 로그와 총 출고량을 서버에서 계산하고, AJAX 출고 후에도 같은 범위를 유지한다.
- 클라이언트 검색/카테고리/구분 필터는 URL 상태와 hidden field로 유지되며, 필터 변경 시 페이지 번호를 1로 재설정한다.
- 컨트롤러 테스트는 DOM id와 모델 계약 중심으로 보강해 인코딩 환경 차이에 덜 민감하게 정리했다.

## 리뷰 포인트
- `item-use`는 출고 실행 화면이지만 상단 로그 영역을 통해 날짜 기반 출고 내역 조회도 함께 담당한다는 점
- 서버는 날짜 범위만 처리하고, 나머지 텍스트/버튼 필터는 클라이언트가 계속 담당한다는 역할 분리
- 초기화 버튼이 전체 무필터 조회가 아니라 `오늘 ~ 오늘` 기본값 복귀라는 점
