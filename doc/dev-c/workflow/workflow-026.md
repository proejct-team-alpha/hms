# 규칙 등록 경로 통일 및 저장 명세서 (workflow-026)

## 문제 정의
관리자 규칙 등록 기능 `S31`을 완성한다. 관리자 규칙 등록 화면과 저장 경로를 `/admin/rule/new`로 통일하고, 제목, 내용, 카테고리, 활성 여부를 입력받아 규칙을 저장할 수 있어야 한다.

## 대상 URL
- `GET /admin/rule/new`
- `POST /admin/rule/new`

## 관련 기존 경로
- `GET /admin/rule/form`
- `POST /admin/rule/form`

## 구현 범위
1. 규칙 등록 화면 경로를 `/admin/rule/new`로 통일
2. 규칙 등록 저장 처리를 `POST /admin/rule/new`로 통일
3. 제목, 내용, 카테고리, 활성 여부 입력 폼 제공
4. 저장 성공 시 목록 리다이렉트 + 성공 메시지 표시
5. 검증 실패 시 동일 화면 재렌더링 + 입력값 유지 + 오류 메시지 표시
6. 기존 `/admin/rule/form` 진입은 한동안 `/admin/rule/new`로 리다이렉트 처리
7. 규칙 목록 화면의 등록 링크를 `/admin/rule/new`로 변경

## 입력 필드
- `title`
  - 필수
  - 최대 200자
- `content`
  - 필수
  - 최대 3000자
- `category`
  - 필수
  - `EMERGENCY`, `SUPPLY`, `DUTY`, `HYGIENE`, `OTHER`
- `active`
  - 체크박스 입력
  - 기본값 `true`
  - 체크 해제 시 `false`

## 저장 이후 흐름
- 저장 성공 시 `redirect:/admin/rule/list`
- 목록 화면에서 성공 플래시 메시지 표시
- 성공 메시지 문구는 기본적으로 `규칙이 등록되었습니다.`를 사용한다.

## 검증/실패 처리
- 제목 미입력 시 오류 메시지 표시
- 제목 200자 초과 시 오류 메시지 표시
- 내용 미입력 시 오류 메시지 표시
- 내용 3000자 초과 시 오류 메시지 표시
- 카테고리 미선택 시 오류 메시지 표시
- 검증 실패 시 `admin/rule/new` 화면을 다시 렌더링한다.
- 검증 실패 시 사용자가 입력한 제목, 내용, 카테고리, 활성 여부를 유지한다.

## 기대 화면 성격
- 관리자 전용 SSR 등록 화면
- 목록 화면과 동일한 관리자 레이아웃 패턴 유지
- 신규 생성 화면이지만 기존 규칙 등록 폼 자산은 버리지 않고 재사용
- 템플릿 이름은 `admin/rule-new`로 통일하고 기존 등록 템플릿 자산을 `rule-new`로 옮겨 재사용

## 제약 조건
- 관리자 영역 URL prefix(`/admin/**`)를 유지한다.
- 컨트롤러/뷰 기준 표준 경로는 `/admin/rule/new`로 통일한다.
- 기존 `/admin/rule/form`은 즉시 제거하지 않고 호환용 리다이렉트 경로로 잠시 유지할 수 있다.
- 새 링크와 새 구현 기준은 모두 `/admin/rule/new`를 사용한다.
- `HospitalRule` 엔티티의 `title` 길이 제한 200자를 따른다.
- 로컬 문서상 기존 `POST /admin/rule/create` 계약이 있으나, 이번 구현 워크플로우에서는 사용자 결정에 따라 `POST /admin/rule/new`로 통일한다.

## 엣지 케이스
- 활성 체크박스가 체크되지 않은 경우 `false`로 저장된다.
- 검증 실패 후에도 체크박스 상태가 유지되어야 한다.
- 구 경로 `/admin/rule/form`으로 접근해도 신규 경로로 자연스럽게 이동해야 한다.
- 규칙 목록 화면에서 등록 버튼을 눌렀을 때 항상 `/admin/rule/new`로 이동해야 한다.

## 구현 순서
1. 현재 `AdminRuleController`, `AdminRuleService`, 규칙 등록 템플릿, `rule-list.mustache` 구조 점검
2. 등록 요청 DTO와 검증 규칙 설계
3. `GET/POST /admin/rule/new` 경로 구현
4. 검증 실패 재렌더링 및 입력값 유지 처리
5. 성공 플래시 메시지 및 목록 리다이렉트 처리
6. 기존 `/admin/rule/form` 호환 리다이렉트 처리
7. 목록 화면 링크를 `/admin/rule/new`로 교체
8. 컨트롤러/서비스 테스트 및 문서 정리

## 리뷰 포인트
- 기존 `/admin/rule/create` 문서 계약을 이번 구현 결정에 맞춰 함께 정리할지

- 검증 실패 시 재렌더링 모델 구성이 현재 관리자 폼 화면 패턴과 잘 맞는지
