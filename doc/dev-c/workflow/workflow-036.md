# Workflow 036 - 직원 수정 화면 퇴사 일시 UX 및 비활성 직원 읽기 전용 정리

## 문제 정의
직원 등록/수정 화면의 `퇴사 일시` 입력이 분 단위까지 요구되어 불필요하게 복잡하다. 또한 비활성화된 직원은 조회는 가능하되 수정은 불가능해야 하는데, 현재는 수정 흐름과 화면 표현이 충분히 분리되어 있지 않다. 이번 작업에서는 `퇴사 일시` 입력 UX를 `날짜 + 시` 기준으로 단순화하고, 비활성 직원은 수정 화면에서 읽기 전용으로만 보이도록 정리한다.

## 적용 범위
- 직원 등록 화면
- 직원 수정 화면
- 직원 수정 SSR 저장
- 관리자 직원 수정 API

## 핵심 요구사항
### 1. 퇴사 일시 입력 단순화
- `퇴사 일시`는 분 단위 없이 `날짜 + 시`까지만 입력한다.
- 입력 UI는 `datetime-local` 대신 아래처럼 분리한다.
  - 날짜 선택 input
  - 시간(`00~23`) select
- 저장 시 분은 항상 `00`으로 고정한다.
  - 예: `2026-03-24` + `15시` 선택 시 실제 저장값은 `2026-03-24T15:00`

### 2. 직원 수정 화면 레이아웃 조정
- 직원 수정 화면에서는 `재직 상태` 옆에 `퇴사 일시`를 나란히 배치한다.
- 두 필드는 한 줄에서 함께 보여, 상태와 퇴사 예약 시점을 직관적으로 확인할 수 있게 한다.

### 3. 직원 등록 화면 반영
- 직원 등록 화면은 현재 레이아웃을 크게 바꾸지 않는다.
- 대신 `퇴사 일시` 입력 형식만 수정 화면과 같은 `날짜 + 시간` 방식으로 단순화한다.

### 4. 비활성 직원 읽기 전용 처리
- 비활성 직원도 수정 화면 진입은 가능하다.
- 하지만 화면은 읽기 전용으로 보여준다.
- 입력창은 `사번`, `로그인 아이디`처럼 회색 비활성 스타일로 렌더링한다.
- 저장 버튼은 노출하지 않는다.
- 화면 상단 안내 문구는 아래 문구를 사용한다.
  - `비활성화된 직원입니다. 조회만 가능합니다.`

### 5. 저장/API 차단
- 비활성 직원 수정은 SSR 저장에서도 차단한다.
- 관리자 직원 수정 API에서도 차단한다.
- 차단 메시지는 아래 문구를 사용한다.
  - `비활성화된 직원은 수정할 수 없습니다.`

## 기대 효과
- `퇴사 일시` 입력 UX가 더 단순해진다.
- 수정 화면에서 `재직 상태`와 `퇴사 일시` 관계를 더 직관적으로 확인할 수 있다.
- 비활성 직원은 조회만 가능하게 해 상태 규칙이 더 명확해진다.
- 화면/SSR/API 규칙이 일관된다.
