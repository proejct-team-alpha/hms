# Admin 메뉴/라우트 연결 명세서 (workflow-003)

## 문제 정의
관리자 사이드바 메뉴 구조가 목표안과 달라서 `/admin/reception/list` 진입 여부를 확인하기 어렵다.
우선 1차로 데이터 로직 없이 메뉴/라우트/화면 진입 연결만 맞춘다.

## 대상 사용자
- 관리자(ROLE_ADMIN): 관리자 메뉴에서 예약/접수 화면 이동
- 개발자: 메뉴 구조 정렬 및 라우트 유효성 확인

## 핵심 요구사항 (우선순위순)
1. [P0] 현재 코드에서 `/admin/reception/list`가 들어갈 수 있는 구조인지 먼저 진단
2. [P0] 진단 후 수정 진행
3. [P0] 1차 작업은 연결만 처리
   - 사이드바 메뉴 링크 연결
   - 컨트롤러 라우트 연결
   - 화면 진입 가능 상태 확보
4. [P1] 데이터 조회(Service/Repository)는 2차로 분리

## 제약 조건 & 전제
- 1차에서는 비즈니스 데이터 로직 추가하지 않음
- 목표는 "메뉴 클릭 → URL 진입 → 화면 렌더링" 확인
- 기존 관리자 레이아웃(L2/S4) 패턴 유지

## 엣지 케이스 & 에러 시나리오
- 사이드바 링크는 추가됐지만 컨트롤러 매핑 누락
- 컨트롤러 매핑은 있으나 템플릿 경로 누락
- 권한 설정으로 접근 차단(ROLE_ADMIN 정책 확인 필요)

## 범위 밖 (명시적 제외)
- `/admin/reception/list` 실제 목록 조회/필터/페이징 구현
- 접수 상태 비즈니스 규칙 정의
- Repository/Service 확장

## 수용 기준
- [ ] 사이드바에 "전체 접수 목록" 메뉴가 보인다.
- [ ] 해당 메뉴 클릭 시 `/admin/reception/list`로 이동된다.
- [ ] 페이지가 에러 없이 렌더링된다(데이터 없이 뼈대/placeholder 허용).
- [ ] 데이터 로직은 포함되지 않는다.

## 구현 순서
1. 관리자 사이드바 템플릿에서 메뉴 구조를 진단하고 "전체 접수 목록" 링크 추가 여부 확인
2. `/admin/reception/list` 라우트 매핑 존재 여부 진단
3. 라우트가 없으면 관리자 컨트롤러에 GET 매핑 추가
4. 템플릿 경로가 없으면 최소 렌더링 가능한 화면(placeholder) 연결
5. 메뉴 클릭 동선(사이드바 → URL → 템플릿 렌더링) 수동 검증
6. 데이터 로직은 제외하고 2차 작업 항목으로 분리 기록
