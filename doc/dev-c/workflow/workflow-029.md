# 규칙 목록 상세 진입 및 삭제 UX 개선 명세서 (workflow-029)

## 문제 정의
관리자 규칙 목록 화면의 사용성을 개선한다.  
현재 목록 화면에서는 상세 화면 진입 UX가 충분히 드러나지 않고, 이미 구현된 규칙 삭제 API `POST /admin/api/rules/{id}`도 목록 화면에 연결되어 있지 않다.  
따라서 규칙 리스트의 각 행 전체를 클릭하면 상세 화면 `GET /admin/rule/detail?ruleId={id}`로 이동하게 하고, 같은 행 안에는 별도 `삭제` 버튼을 두어 AJAX로 삭제를 처리한 뒤 목록은 유지한 채 해당 행만 제거되도록 만든다.

## 대상 URL
- `GET /admin/rule/list`
- `GET /admin/rule/detail?ruleId={id}`
- `POST /admin/api/rules/{id}`

## 비채택 경로 / 방식
- 제목 링크만 클릭해서 상세로 이동하는 UX 유지
- 삭제 후 전체 페이지 리로드
- 삭제 후 목록 페이지 재이동
- 커스텀 모달 기반 삭제 확인
- soft delete / active=false 전환

## 구현 범위
1. 규칙 목록의 각 행 전체를 클릭하면 `GET /admin/rule/detail?ruleId={id}`로 이동하도록 개선
2. 제목 링크 중심 UX를 행 전체 클릭 UX로 전환
3. 각 행에 별도 `삭제` 버튼 추가
4. 삭제 버튼 클릭 시 행 클릭 이벤트와 충돌하지 않도록 이벤트 전파 분리
5. 삭제 버튼은 기존 관리자 JSON API `POST /admin/api/rules/{id}` 호출
6. 삭제 성공 시 현재 목록을 유지하면서 해당 행만 DOM에서 제거
7. 삭제 성공 시 성공 toast 메시지 표시
8. 삭제 실패 시 실패 toast 메시지 표시
9. 마지막 행 삭제 시 empty state 문구가 자연스럽게 보이도록 처리

## 상세 UX 계약

### 상세 진입
- 목록 테이블의 각 규칙 행은 hover 시 현재처럼 배경이 살짝 회색으로 변한다.
- 그 상태에서 행의 비삭제 영역을 클릭하면 상세 화면으로 이동한다.
- 이동 URL은 기존 계약을 유지한다.
  - `GET /admin/rule/detail?ruleId={id}`

### 삭제 버튼
- 삭제 버튼은 각 행 내부의 별도 액션 영역에 둔다.
- 삭제 버튼 클릭 시:
  1. `confirm()`으로 삭제 여부 확인
  2. 확인 시 `POST /admin/api/rules/{id}` 호출
  3. 성공 응답이면 해당 행만 제거
  4. 성공 toast 메시지 표시
  5. 실패 시 실패 toast 메시지 표시
- 삭제 버튼 클릭은 행 전체 클릭 상세 이동을 트리거하면 안 된다.

### 목록 유지 정책
- 삭제 성공 후 페이지 이동은 없다.
- 필터, 검색어, 현재 페이지 상태는 유지한다.
- DOM에서 삭제된 규칙 행만 제거한다.
- 남은 행이 없으면 “조회된 규칙이 없습니다.” empty state를 보여준다.

## 응답 연동 계약
- 삭제 API는 기존 구현을 그대로 사용한다.
- 엔드포인트:
  - `POST /admin/api/rules/{id}`
- 성공 응답 형식:
  - `Resp.ok(...)`
- 성공 body:
  - `ruleId`
  - `message`
- 기본 성공 메시지:
  - `규칙이 삭제되었습니다.`

## 실패 처리
- 삭제 실패 시 목록 화면 내 toast 영역에서 실패 메시지를 보여준다.
- 별도 alert 창은 사용하지 않는다.
- 실패 시 현재 목록 상태는 유지한다.
- 주요 실패 케이스:
  - 대상 규칙이 이미 삭제되었거나 존재하지 않음
  - 권한 없음
  - 네트워크/서버 오류

## 제약 조건
- 상세 URL 계약은 그대로 `GET /admin/rule/detail?ruleId={id}` 유지
- 삭제 API는 새로 만들지 않고 기존 `POST /admin/api/rules/{id}` 재사용
- 삭제 확인은 브라우저 기본 `confirm()` 사용
- 후속 피드백은 toast 기반
- SSR 목록 화면에 최소한의 JS만 추가해 동작시키는 방향으로 구현
- 기존 hover 스타일과 관리자 목록 레이아웃은 최대한 유지

## 엣지 케이스
- 삭제 버튼 클릭 시 이벤트 버블링으로 상세 이동이 같이 일어나면 안 된다.
- 이미 삭제된 규칙을 다시 삭제하려는 경우 실패 toast를 보여준다.
- 삭제 성공 후 현재 화면에 남은 데이터가 0건이면 empty state로 전환한다.
- 권한 없는 사용자의 API 호출은 공통 실패 응답을 받아 실패 toast로 표현한다.
- 검색/필터 결과가 1건일 때 삭제하면 목록 테이블 구조가 깨지지 않아야 한다.
