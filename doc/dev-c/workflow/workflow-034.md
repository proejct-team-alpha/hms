# Workflow 034 - 관리자 대시보드 그래프 리팩토링

## 문제 정의
관리자 대시보드의 그래프 2개를 운영 화면 기준에 맞게 리팩토링한다.

1. `일일 환자수 추이`
   - 데이터 계산 방식은 유지한다.
   - 시각 스타일만 원무과 대시보드의 `시간대별 예약 및 접수 현황` 그래프 톤에 맞춘다.
2. `카테고리별 물품 재고 현황`
   - 기존 카테고리별 재고 비교 그래프를 제거한다.
   - 물품 담당자 대시보드의 `최근 7일 입고/출고 추이`와 같은 기준으로 교체한다.

## 기준 화면
- 원무과 대시보드: `시간대별 예약 및 접수 현황`
- 물품 담당자 대시보드: `최근 7일 입고/출고 추이`

## 구현 범위
### 1. 관리자 환자 그래프
- `dailyPatients` 데이터 계약은 그대로 유지한다.
- 카드 레이아웃, 색감, 축/격자선, 범례 톤을 원무과 그래프 스타일에 맞춘다.
- 최근 7일 배지와 설명 문구를 추가해 카드 밀도를 맞춘다.

### 2. 관리자 물품 그래프
- 기존 `categoryCounts` 기반 그래프는 제거한다.
- 최근 7일 `입고 / 출고` 2시리즈 그래프로 교체한다.
- 기간, 축, 데이터 계산 방식은 물품 담당자 대시보드와 동일 기준을 따른다.

### 3. 테스트 및 문서
- 관리자 대시보드 SSR/API/서비스 테스트를 새 그래프 구조 기준으로 갱신한다.
- 구현 결과와 검증 범위를 workflow/task/report에 반영한다.

## 기대 효과
- 관리자 대시보드 그래프 UI 톤이 다른 운영 화면과 일관된다.
- 물품 그래프가 재고 스냅샷보다 실제 입출고 흐름을 더 잘 보여준다.
- 관리자 화면에서 최근 환자 흐름과 최근 물품 흐름을 함께 읽기 쉬워진다.

## 수용 기준
- [x] 관리자 대시보드 `일일 환자수 추이` 그래프가 원무과 그래프와 같은 톤으로 렌더링된다.
- [x] 해당 그래프는 기존 데이터 계산 방식을 유지한다.
- [x] 관리자 대시보드의 기존 `카테고리별 물품 재고 현황` 그래프가 제거된다.
- [x] 대신 최근 7일 기준 `입고 / 출고` 2시리즈 그래프가 렌더링된다.
- [x] 물품 담당자 화면과 기간/축/데이터 계산 방식이 일치한다.
- [x] 관련 관리자 대시보드 테스트가 통과한다.

## 최종 검증 메모
- 범위 테스트: `./gradlew cleanTest test --tests 'com.smartclinic.hms.admin.dashboard.*'` 통과
- 전체 테스트: `./gradlew test` 실행 시 `AdminItemControllerTest > history uses today as default date range` 1건 실패
- 위 전체 테스트 실패는 현재 대시보드 그래프 리팩토링 범위가 아니라 `admin.item` 테스트 문자열 기대값 불일치로 확인했다.
