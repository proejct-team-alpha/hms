# 관리자 대시보드 체크리스트 (W1 기준)

## A. 4개 통계 구현 (DB 조회)
- [ ] 오늘 예약 건수가 DB 기준으로 표시된다.
- [ ] 총 예약 건수가 DB 기준으로 표시된다.
- [ ] 활성 직원 수가 DB 기준으로 표시된다.
- [ ] 재고 부족 물품 수(`quantity < min_quantity`)가 DB 기준으로 표시된다.
- [ ] 4개 값 모두 하드코딩이 아니라 Repository/Service 조회 결과다.
- [ ] 데이터 0건일 때도 화면이 깨지지 않고 `0`으로 표시된다.

## B. L2 + S4 사이드바 적용
- [ ] 관리자 페이지가 L2 레이아웃(헤더/본문/푸터) 구조를 따른다.
- [ ] 관리자 전용 사이드바(S4)가 렌더링된다.
- [ ] 사이드바 메뉴 항목이 모두 보인다: 대시보드/예약관리/환자관리/인사관리/진료과관리/물품관리/규칙관리
- [ ] 각 메뉴 링크가 정상 이동한다.
- [ ] 현재 페이지 메뉴 active 표시가 정상이다.
- [ ] 모바일/축소 화면에서도 레이아웃이 깨지지 않는다.

## C. 최종 검증
- [ ] `/admin/dashboard` 접속 시 에러 없이 로드된다.
- [ ] 권한 없는 계정 접근 시 차단된다(ROLE_ADMIN 전용).
- [ ] 대시보드 새로고침 시 통계 값이 일관된다.
- [ ] 콘솔 에러(JS) 없이 렌더링된다.
