# 규칙 목록 완성 명세서 (workflow-025)

## 문제 정의
관리자 규칙 목록 화면 `GET /admin/rule/list`를 검색형 목록 화면으로 완성한다. 카테고리 필터, 활성 여부 필터, 제목 키워드 검색, 페이지네이션, 활성 상태 배지 표시를 지원한다.

## 대상 URL
- `GET /admin/rule/list`

## 구현 범위
1. 카테고리 필터
2. 활성 여부 필터
3. 제목 키워드 검색
4. 페이지네이션
5. 목록에서 활성 상태 배지 표시

## 검색/필터 기준
- `category`
  - 기본값: `ALL`
- `active`
  - 기본값: `ALL`
- `keyword`
  - optional
  - 제목 기준 `LIKE` 검색
- `page`
  - 기본값: `1`
- `size`
  - 기본값: `10`

## 정렬 기준
- 관리자 목록 화면 성격에 맞게 최신 등록순으로 정렬한다.
- 우선 `id desc` 또는 `createdAt desc` 기준으로 구현한다.

## 기대 화면 성격
- 통계형 대시보드가 아니라 관리용 검색형 목록 화면으로 구성한다.
- 즉, “찾고, 필터링하고, 상태를 확인하는 인덱스 화면”으로 본다.

## 제약 조건
- 기존 관리자 영역 URL prefix(`/admin/**`)를 유지한다.
- 기존 규칙 관리 레이아웃 패턴과 관리자 목록 페이지네이션 패턴을 유지한다.
- SSR 목록 화면 기준으로 구현한다.
- 검색은 우선 제목 기준만 적용한다.
- 필터/검색/페이지네이션 조합 시 현재 조건을 유지한 채 이동할 수 있어야 한다.

## 엣지 케이스
- `category`, `active`가 비어 있거나 잘못된 값일 때는 기본값(`ALL`)로 안전하게 처리한다.
- `keyword`가 공백만 들어온 경우는 검색어 없음으로 처리한다.
- 검색 결과가 없더라도 필터 상태와 페이지 영역은 일관되게 렌더링한다.
- 필터 조합 상태에서 페이지 이동 시 현재 조건이 유지되어야 한다.

## 수용 기준
- [ ] `GET /admin/rule/list`에서 규칙 목록이 페이지 단위로 조회된다.
- [ ] 카테고리 필터가 동작한다.
- [ ] 활성 여부 필터가 동작한다.
- [ ] 제목 키워드 검색이 동작한다.
- [ ] 목록에 활성 상태 배지가 표시된다.
- [ ] 필터/검색 조건을 유지한 채 페이지 이동이 가능하다.
- [ ] 관련 컨트롤러/서비스/리포지토리 테스트가 통과한다.

## 구현 순서
1. 현재 규칙 목록 컨트롤러/서비스/리포지토리 구조 점검
2. 목록 응답 DTO와 페이지 링크 구조 설계
3. 카테고리/활성/제목 검색 + 페이지네이션 구현
4. 규칙 목록 Mustache 화면 정리
5. 컨트롤러/서비스 테스트 보강
6. 문서와 검증 결과 정리
